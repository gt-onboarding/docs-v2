<ParamField body="MfaPollingOptions" type="object">
  Options de configuration pour gérer un processus d’interrogation par notifications push AMF.

  Fournissez ces options au démarrage d’une session d’interrogation pour contrôler la fréquence
  des requêtes vers le endpoint de défi et la façon dont la réussite ou l’échec sont gérés.

  <Expandable title="properties">
    <ParamField body="intervalMs?" type="number">
      Intervalle facultatif, en **millisecondes**, entre des requêtes d’interrogation consécutives.

      * S’il est omis, l’intervalle par défaut interne du SDK est utilisé (généralement 5000 ms).
      * Choisissez une valeur qui équilibre la réactivité et les considérations de limitation de débit.

      ```ts Valeur par défaut
      Spécifique à l’implémentation (généralement 5000 ms)
      ```
    </ParamField>

    <ParamField body="onCompleted()?" type="void">
      Fonction de rappel facultative exécutée une fois que le défi push AMF a été approuvé
      avec succès et que l’interrogation est terminée.

      * Appelée exactement **une fois**, après quoi l’interrogation s’arrête automatiquement.
      * Utile pour faire progresser le flux de connexion (par exemple, en appelant `continue()`).
    </ParamField>

    <ParamField body="onError()?" type="void">
      Fonction de rappel facultative invoquée si une erreur se produit pendant l’interrogation.

      Reçoit un objet [ULError](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error) contenant :

      * `status` — le code d’état HTTP de la requête ayant échoué

      * `responseText` — le corps de la réponse brute, s’il est disponible

      * Appelée une fois par événement d’erreur ; l’interrogation peut réessayer selon le type d’erreur.

      * Utilisez-la pour afficher des messages d’erreur ou déclencher une logique personnalisée de nouvelle tentative ou d’abandon.

      **Propriétés**

      <ParamField body="error" type={<a href="/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error">Error</a>} />
    </ParamField>
  </Expandable>
</ParamField>
