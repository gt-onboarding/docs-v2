    <ParamField body="options" type={<span><a href="/docs/libraries/acul/react-sdk/API-Reference/Types/type-aliases/MfaPollingOptions">MfaPollingOptions</a></span>}>
      <ParamField body="MfaPollingOptions" type="object">
        Options de configuration pour gérer un processus d’interrogation (polling) par notification push d’AMF (MFA).

        Fournissez ces options au démarrage d’une session d’interrogation pour contrôler la fréquence
        d’appel du point de terminaison du défi et la façon dont les réussites ou les échecs sont gérés.

        <Expandable title="propriétés">
          <ParamField body="intervalMs?" type="number">
            Intervalle facultatif, en **millisecondes**, entre deux requêtes d’interrogation consécutives.

            * S’il est omis, l’intervalle interne par défaut du SDK est utilisé (généralement 5000 ms).
            * Choisissez une valeur qui équilibre la réactivité et les considérations liées aux limites de débit (rate limit).
          </ParamField>

          <ParamField body="onCompleted()?" type="void">
            Rappel facultatif exécuté lorsque le défi push d’AMF (MFA) est approuvé
            avec succès et que l’interrogation est terminée.

            * Appelé exactement **une seule fois**, après quoi l’interrogation s’arrête automatiquement.
            * Utile pour faire avancer le flux de connexion (par exemple, en appelant `continue()`).
          </ParamField>

          <ParamField body="onError()?" type="void">
            Rappel facultatif invoqué si une erreur survient pendant l’interrogation.

            Reçoit un objet [ULError](/docs/fr-CA/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error) contenant :

            * `status` — le code d’état HTTP de la requête ayant échoué

            * `responseText` — le corps de la réponse brute, s’il est disponible

            * Appelé une fois par événement d’erreur; l’interrogation peut tenter de nouveau selon le type d’erreur.

            * À utiliser pour afficher des messages d’erreur ou déclencher une logique personnalisée de nouvelle tentative ou d’abandon.
