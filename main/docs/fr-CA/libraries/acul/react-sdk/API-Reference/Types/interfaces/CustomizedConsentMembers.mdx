CustomizedConsentMembers

````ts Example
export interface CustomizedConsentMembers extends BaseMembers {
  /**
   * Provides access to the specific properties and data of the Customized Consent screen,
   * including the list of `scopes` and `authorizationDetails` being requested.
   * @type {ScreenMembersOnCustomizedConsent}
   */
  screen: ScreenMembersOnCustomizedConsent;

  /**
   * Soumet la décision de l'utilisateur d'accepter (accorder) les permissions et détails d'autorisation demandés.
   * Cette action envoie une requête POST au point de terminaison `/u/customized-consent` avec `action: "accept"`.
   * L'état de la transaction est automatiquement inclus.
   *
   * @param {CustomOptions} [payload] - Optionnel. Un objet pour toute paire clé-valeur personnalisée
   *                                    à envoyer avec la requête. Ces paramètres seront
   *                                    inclus dans les données de formulaire soumises au serveur.
   * @returns {Promise<void>} Une promesse qui se résout lorsque l'action d'acceptation est soumise avec succès.
   *                          En cas de succès, Auth0 redirige généralement l'utilisateur vers l'application
   *                          ou vers la prochaine étape du flux d'authentification.
   * @throws {Error} Lance une erreur si la soumission du formulaire échoue (p. ex., problème réseau, état invalide).
   *                 Les erreurs côté serveur (comme « invalid_request ») seront reflétées dans `this.transaction.errors`
   *                 après l'opération, plutôt que d'être lancées comme erreurs JavaScript.
   *
   * @example
   * ```typescript
   * // En supposant que 'customizedConsentManager' est une instance de la classe SDK CustomizedConsent
   * try {
   *   await customizedConsentManager.accept();
   *   // En cas de succès, la page redirigera généralement.
   * } catch (error) {
   *   // Gérer les erreurs inattendues lors de la soumission elle-même.
   *   console.error("Échec de la soumission de l'acceptation du consentement :", error);
   * }
   * // Après l'await, vérifier customizedConsentManager.transaction.errors pour les problèmes de validation côté serveur.
   * ```
   */
  accept(payload?: CustomOptions): Promise<void>;

  /**
   * Submits the user's decision to deny (reject) the requested permissions and authorization details.
   * This action posts to the `/u/customized-consent` endpoint with `action: "deny"`.
   * The transaction state is automatically included.
   *
   * @param {CustomOptions} [payload] - Optional. An object for any custom key-value pairs
   *                                    to be sent with the request. These parameters will be
   *                                    included in the form data submitted to the server.
   * @returns {Promise<void>} A promise that resolves when the deny action is successfully submitted.
   *                          On success, Auth0 typically redirects the user, potentially showing an
   *                          access denied message or returning an error to the application.
   * @throws {Error} Throws an error if the form submission fails (e.g., network issue, invalid state).
   *                 Server-side errors are reflected in `this.transaction.errors`.
   *
   * @example
   * ```typescript
   * // Assuming 'customizedConsentManager' is an instance of the CustomizedConsent SDK class
   * try {
   *   await customizedConsentManager.deny({ reason_code: "user_rejected_details" }); // Example custom option
   *   // If successful, the page will typically redirect.
   * } catch (error) {
   *   console.error("Failed to submit consent denial:", error);
   * }
   * // After the await, check customizedConsentManager.transaction.errors for server-side validation issues.
   * ```
   */
  deny(payload?: CustomOptions): Promise<void>;
}
````

<ParamField body='accept' type='Promise<void>'>

Soumet la décision de l'utilisateur d'accepter (accorder) les permissions et les détails d'autorisation demandés.
Cette action envoie une requête vers le point de terminaison `/u/customized-consent` avec `action: "accept"`.
L'état de la transaction est automatiquement inclus.

Une promesse qui se résout lorsque l'action d'acceptation est soumise avec succès.
                         En cas de réussite, Auth0 redirige généralement l'utilisateur vers l'application
                         ou vers l'étape suivante du flux d'authentification.

<div id="throws">
  #### Lève
</div>

Lève une erreur si l'envoi du formulaire échoue (p. ex., problème réseau, `state` non valide).
                Les erreurs côté serveur (comme "invalid_request") seront reflétées dans `this.transaction.errors`
                après l'opération, plutôt que d'être levées comme erreurs JavaScript.

```typescript Example
// En supposant que 'customizedConsentManager' soit une instance de la classe CustomizedConsent du SDK
try {
  await customizedConsentManager.accept();
  // En cas de réussite, la page sera généralement redirigée.
} catch (error) {
  // Gérer les erreurs inattendues lors de l'envoi lui-même.
  console.error("Échec de l'envoi de l'acceptation du consentement :", error);
}
// Après l'instruction await, vérifiez customizedConsentManager.transaction.errors pour les problèmes de validation côté serveur.
```

<ParamField body='deny' type='Promise<void>'>

Soumet la décision de l'utilisateur de refuser (rejeter) les permissions et les détails d'autorisation demandés.
Cette action envoie une requête vers le point de terminaison `/u/customized-consent` avec `action: "deny"`.
L'état de la transaction est automatiquement inclus.

Une promesse qui se résout lorsque l'action de refus est soumise avec succès.
                         En cas de réussite, Auth0 redirige généralement l'utilisateur, en affichant éventuellement
                         un message d'accès refusé ou en renvoyant une erreur à l'application.

<div id="throws">
  #### Lève
</div>

Lève une erreur si l'envoi du formulaire échoue (p. ex., problème réseau, `state` non valide).
                Les erreurs côté serveur sont reflétées dans `this.transaction.errors`.

```typescript Example
// En supposant que 'customizedConsentManager' soit une instance de la classe CustomizedConsent du SDK
try {
  await customizedConsentManager.deny({ reason_code: "user_rejected_details" }); // Exemple d'option personnalisée
  // En cas de réussite, la page sera généralement redirigée.
} catch (error) {
  console.error("Échec de l'envoi du refus de consentement :", error);
}
// Après l'instruction await, vérifiez customizedConsentManager.transaction.errors pour les problèmes de validation côté serveur.
```
