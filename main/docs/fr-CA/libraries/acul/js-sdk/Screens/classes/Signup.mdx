<ParamField body="federatedSignup" type="Promise<void>">
  Gère l&#39;envoi du formulaire d&#39;inscription via un fournisseur social.

  #### Remarques

  Cette méthode est similaire à la méthode [signup](#signup), mais elle est utilisée pour les inscriptions via un fournisseur social.

  ```typescript Example
  import Signup from '@auth0/auth0-acul-js/signup';

  const signupManager = new Signup();

  signupManager.federatedSignup({
   connection: 'google-oauth2'
  });
  ```

  <Expandable title="Paramètres">
    <ParamField body="payload" type={<span><a href="/docs/libraries/acul/js-sdk/Screens/interfaces/FederatedSignupPayloadOptions">FederatedSignupPayloadOptions</a></span>}>
      **Propriétés**

      <ParamField body="connection" type="string" />
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="getErrors" type={<span><a href="/docs/libraries/acul/js-sdk/Screens/interfaces/Error">Error</a>[]</span>}>
  Récupère, à partir du contexte, le tableau des erreurs de la transaction ou un tableau vide si aucune erreur n&#39;existe.

  Un tableau d&#39;objets d&#39;erreur provenant du contexte de transaction.
</ParamField>

<ParamField body="getSignupIdentifiers" type={<span><a href="/docs/libraries/acul/js-sdk/Screens/interfaces/Identifier">Identifier</a>[]</span>}>
  **`Utility Feature`**

  Renvoie la liste des identifiants activés pour le formulaire d&#39;inscription,
  en indiquant pour chacun s&#39;il est obligatoire ou facultatif selon la configuration de la transaction.

  Tableau d&#39;objets d&#39;identifiant (par exemple, courriel, téléphone, nom d&#39;utilisateur).

  ```ts Example
  const signup = new Signup();
  const identifiers = signup.getSignupIdentifiers();
  // [{ type: 'email', required: true }, { type: 'username', required: false }]
  ```
</ParamField>

<ParamField body="validateUsername" type={<span><a href="/docs/libraries/acul/js-sdk/Screens/interfaces/UsernameValidationResult">UsernameValidationResult</a></span>}>
  **`Utility Feature`**

  Valide un nom d&#39;utilisateur donné par rapport à la stratégie de noms d&#39;utilisateur actuelle
  définie dans le contexte de transaction.

  Objet de résultat indiquant si le nom d&#39;utilisateur est valide et pourquoi.

  ```ts Example
  const signup = new Signup();
  const result = signup.validateUsername('myusername');
  // result => { valid: true, errors: [] }
  ```

  <Expandable title="Paramètres">
    <ParamField body="username" type="string">
      `string`

      La chaîne de caractères du nom d&#39;utilisateur à valider.
    </ParamField>
  </Expandable>
</ParamField>
