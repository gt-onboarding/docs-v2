<div id="methods">
  ## Méthodes
</div>

<ParamField body='continue' type='Promise<void>'>

Permet à l’utilisateur de poursuivre le processus d’authentification après une inscription WebAuthn réussie.
Cette méthode envoie une requête POST au point de terminaison `/u/mfa-webauthn-enrollment-success`
avec l’action définie à « default », ce qui indique que l’utilisateur passe à partir de cet écran de réussite.

Une promesse qui est résolue lorsque l’action de poursuite est soumise avec succès.
                         En cas de soumission réussie, le serveur Auth0 redirigera généralement l’utilisateur
                         vers l’étape suivante appropriée du processus d’authentification.

<div id="throws">
  #### Lève
</div>

Lève une erreur si l’envoi du formulaire échoue (par exemple, en raison de problèmes réseau,
                d’un `state` de transaction non valide ou d’autres erreurs de validation côté serveur).
                Si une erreur se produit, des détails peuvent être disponibles dans `this.transaction.errors`
                si la page est rendue de nouveau.

```typescript Example
// En supposant que « sdk » est une instance de MfaWebAuthnEnrollmentSuccess
async function handleContinueClick() {
  try {
    await sdk.continue();
    // L’utilisateur sera redirigé par Auth0.
  } catch (error) {
    console.error("Erreur lors de la poursuite après une inscription WebAuthn réussie :", error);
    // Affichez un message d’erreur à l’utilisateur, potentiellement à partir de sdk.transaction.errors
  }
}
```
