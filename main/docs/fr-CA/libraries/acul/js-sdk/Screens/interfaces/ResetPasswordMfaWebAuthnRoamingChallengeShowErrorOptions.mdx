ShowErrorOptions

```ts Example
export interface ShowErrorOptions {
  /**
   * L'objet d'erreur capturé lors d'un appel `navigator.credentials.get()` ayant échoué.
   * Celui-ci doit inclure au minimum les propriétés `name` et `message` de la DOMException.
   * @type {WebAuthnErrorDetails}
   * @example { name: "NotAllowedError", message: "The operation was aborted by the user." }
   */
  error: WebAuthnErrorDetails;

  /**
   * Facultatif. Si `true`, et si le contexte de l'écran (`screen.showRememberDevice`)
   * indique que la mémorisation de l'appareil est une option, ceci signale l'intention
   * de l'utilisateur de mémoriser ce navigateur/appareil.
   * @type {boolean | undefined}
   */
  rememberDevice?: boolean;
}
```

<ParamField body='error' type={<span><a href="/docs/libraries/acul/js-sdk/Screens/interfaces/WebAuthnErrorDetails">WebAuthnErrorDetails</a></span>}>
L’objet d’erreur provenant de l’API WebAuthn (`navigator.credentials.get()`)
qui contient les propriétés `name` et `message` de la DOMException.
</ParamField>

<ParamField body='rememberDevice?' type='boolean'>
Facultatif. Si `true` et `screen.showRememberDevice` est également `true`,
la trousse de développement logiciel (SDK) inclura `rememberBrowser=true` dans la soumission du formulaire.
</ParamField>
