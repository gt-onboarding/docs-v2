export interface ResetPasswordMfaWebAuthnRoamingChallengeMembers extends BaseMembers {

  /**
 * Access to the specific properties and data of the `reset-password-mfa-webauthn-roaming-challenge` screen,
 * including WebAuthn `publicKey` challenge options and the `showRememberDevice` flag.
 * @type {ScreenMembersOnResetPasswordMfaWebAuthnRoamingChallenge}
 */
  screen: ScreenMembersOnResetPasswordMfaWebAuthnRoamingChallenge;

  /**
   * Initie le processus d'assertion WebAuthn (défi de clé de sécurité).
   * Cette méthode appellera en interne `navigator.credentials.get()` en utilisant le défi
   * fourni dans `screen.publicKey`.
   * En cas d'interaction réussie avec la clé de sécurité, elle soumet le
   * `PublicKeyCredential` résultant à Auth0 avec `action: "default"`.
   *
   * @param {UseSecurityKeyOptions} [options] - Optionnel. Paramètres pour l'opération,
   * tels que `rememberDevice` (si `screen.showRememberDevice` est vrai) et d'autres options personnalisées.
   * Le champ `response` (l'identifiant WebAuthn) est géré en interne par le SDK.
   * @returns {Promise<void>} Une promesse qui se résout lorsque la tentative de vérification est soumise.
   * Une opération réussie entraîne généralement une redirection par Auth0.
   * @throws {Error} Lance une erreur si `screen.publicKey` est manquant, si `navigator.credentials.get()`
   * échoue (p. ex., annulation par l'utilisateur, aucun authentificateur trouvé), ou si la soumission du formulaire à Auth0 échoue.
   * Il est recommandé d'intercepter ces erreurs et d'utiliser potentiellement `showError()` pour signaler les problèmes spécifiques à l'API WebAuthn.
   */
  useSecurityKey(options?: UseSecurityKeyOptions): Promise<void>;

  /**
   * Reports a client-side WebAuthn API error (from `navigator.credentials.get()`) to Auth0.
   * This method should be used when `useSecurityKey()` (or a manual `navigator.credentials.get()` call)
   * fails due to a browser-side WebAuthn issue (e.g., `NotAllowedError` if the user cancels the prompt,
   * `NotFoundError` if no matching authenticator is found, or a timeout).
   * It submits the error details with `action: "showError::{errorDetailsJsonString}"` and an empty `response`.
   *
   * @param {ShowErrorOptions} options - Contains the `error` object (with `name` and `message`
   * from the WebAuthn API DOMException), an optional `rememberDevice` flag, and any other custom options.
   * @returns {Promise<void>} A promise that resolves when the error report is submitted.
   * Auth0 may re-render the page with error information or redirect.
   * @throws {Error} Throws an error if the form submission fails (e.g., network error, invalid state).
   */
  showError(options: ShowErrorOptions): Promise<void>;

  /**
   * Allows the user to opt-out of the WebAuthn roaming challenge and select a different MFA method.
   * This action submits `action: "pick-authenticator"` to Auth0, which should navigate
   * the user to an MFA factor selection screen.
   *
   * @param {TryAnotherMethodOptions} [options] - Optional. Parameters for the operation,
   * such as `rememberDevice` (if `screen.showRememberDevice` is true) and other custom options.
   * @returns {Promise<void>} A promise that resolves when the 'pick-authenticator' action is submitted.
   * @throws {Error} Throws an error if the form submission fails.
   */
  tryAnotherMethod(options?: TryAnotherMethodOptions): Promise<void>;
}

<ParamField body='showError' type='Promise<void>'>

Signale à Auth0 une erreur WebAuthn de l’API côté client (provenant de `navigator.credentials.get()`).
Cette méthode doit être utilisée lorsque `useSecurityKey()` (ou un appel manuel à `navigator.credentials.get()`)
échoue en raison d’un problème WebAuthn côté navigateur (par exemple, `NotAllowedError` si l’utilisateur annule l’invite,
`NotFoundError` si aucun authentificateur correspondant n’est trouvé, ou un délai d’attente).
Elle envoie les détails de l’erreur avec `action: "showError::{errorDetailsJsonString}"` et une `response` vide.

Une promesse résolue lorsque le rapport d’erreur est soumis.
Auth0 peut réafficher la page avec des informations sur l’erreur ou effectuer une redirection.

<div id="throws">
  #### Lève
</div>

Lève une erreur si la soumission du formulaire échoue (par exemple, erreur réseau, état invalide).

<ParamField body='tryAnotherMethod' type='Promise<void>'>

Permet à l’utilisateur d’abandonner le défi WebAuthn itinérant et de sélectionner une autre méthode d’AMF (MFA).
Cette action soumet `action: "pick-authenticator"` à Auth0, ce qui devrait amener
l’utilisateur vers un écran de sélection du facteur d’AMF.

Une promesse résolue lorsque l’action « pick-authenticator » est soumise.

<ParamField body='useSecurityKey' type='Promise<void>'>

Démarre le processus d’assertion WebAuthn (défi de clé de sécurité).
Cette méthode appellera en interne `navigator.credentials.get()` à l’aide du défi
fourni dans `screen.publicKey`.
Après une interaction réussie avec la clé de sécurité, elle soumet le
`PublicKeyCredential` résultant à Auth0 avec `action: "default"`.

Une promesse résolue lorsque la tentative de vérification est soumise.
Une opération réussie entraîne généralement une redirection par Auth0.

<div id="throws">
  #### Lève
</div>

Lève une erreur si `screen.publicKey` est absent, si `navigator.credentials.get()`
échoue (par exemple, annulation par l’utilisateur, aucun authentificateur trouvé), ou si la soumission du formulaire à Auth0 échoue.
Il est recommandé d’intercepter ces erreurs et d’utiliser éventuellement `showError()` pour signaler les problèmes spécifiques à l’API WebAuthn.
