````ts Example
export interface MfaWebAuthnPlatformChallengeMembers extends BaseMembers {
  /**
   * Screen-specific properties and data, including WebAuthn challenge options and remember device preference.
   * @type {ScreenMembersOnMfaWebAuthnPlatformChallenge}
   */
  screen: ScreenMembersOnMfaWebAuthnPlatformChallenge;

  /**
   * Lance le défi d'authentification de la plateforme WebAuthn.
   * Cette méthode appelle en interne `navigator.credentials.get()` en utilisant les options de défi
   * fournies dans `screen.publicKey`.
   * En cas de succès, elle soumet les informations d'identification obtenues à Auth0 avec `action: "default"`.
   *
   * Si `navigator.credentials.get()` échoue (par exemple, annulation par l'utilisateur, expiration du délai ou autres erreurs de l'API WebAuthn),
   * il est recommandé d'appeler `reportBrowserError` avec les détails de l'erreur.
   *
   * @param {VerifyPlatformAuthenticatorOptions} [options] - Paramètres optionnels pour le processus de vérification,
   * tels que `rememberDevice` (si `screen.showRememberDevice` est true) et d'autres options personnalisées.
   * @returns {Promise<void>} Une promesse qui se résout lorsque la tentative de vérification est soumise.
   *                          Une opération réussie entraîne généralement une redirection.
   * @throws {Error} Lance une erreur si `screen.publicKey` est manquant,
   *                 si `navigator.credentials.get()` échoue avec une erreur inattendue non gérée par `getPasskeyCredentials`,
   *                 ou si la soumission du formulaire à Auth0 échoue.
   *
   * @example
   * ```typescript
   * // En supposant que 'sdk' est une instance de MfaWebAuthnPlatformChallenge
   * try {
   *   const remember = sdk.screen.showRememberDevice && userCheckedRememberDeviceBox;
   *   await sdk.verify({ rememberDevice: remember });
   *   // En cas de succès, Auth0 gère la redirection.
   * } catch (error) {
   *   console.error("La vérification de l'authentificateur de plateforme a échoué :", error);
   *   // S'il s'agit d'une erreur de l'API WebAuthn, signalez-la
   *   if (error.name && error.message) { // Vérification de base pour une erreur de type DOMException
   *     await sdk.reportBrowserError({ error: { name: error.name, message: error.message } });
   *   }
   *   // Vérifiez sdk.transaction.errors pour les messages de validation côté serveur si la page se recharge.
   * }
   * ```
   */
  verify(options?: VerifyPlatformAuthenticatorOptions): Promise<void>;

  /**
   * Reports a specific WebAuthn API error (from `navigator.credentials.get()`) to Auth0.
   * This method should be used when `verify()` (or a manual `navigator.credentials.get()` call) fails due to
   * a browser-side WebAuthn issue (e.g., user cancellation `NotAllowedError`, timeout).
   * It submits the error details with `action: "showError::{errorDetailsJsonString}"` and an empty `response`.
   *
   * @param {ReportBrowserErrorOptions} options - Contains the `error` object (with `name` and `message`
   * from the WebAuthn API DOMException) and any other custom options.
   * @returns {Promise<void>} A promise that resolves when the error report is submitted.
   * @throws {Error} Throws an error if the form submission fails (e.g., network error, invalid state).
   *
   * @example
   * ```typescript
   * // Assuming 'sdk' is an instance of MfaWebAuthnPlatformChallenge
   * // And webAuthnDomException is an error object from a failed navigator.credentials.get() call.
   * try {
   *   await sdk.reportBrowserError({
   *     error: { name: webAuthnDomException.name, message: webAuthnDomException.message }
   *   });
   *   // Auth0 will process this error and may re-render the page or redirect.
   * } catch (submitError) {
   *   console.error("Failed to report WebAuthn browser error:", submitError);
   * }
   * ```
   */
  reportBrowserError(options: ReportBrowserErrorOptions): Promise<void>;

  /**
   * Allows the user to opt-out of the WebAuthn platform challenge and select a different MFA method.
   * This action submits `action: "pick-authenticator"` to Auth0, which should navigate
   * the user to an MFA factor selection screen.
   *
   * @param {TryAnotherMethodOptions} [options] - Optional custom parameters to be sent with the request.
   * @returns {Promise<void>} A promise that resolves when the 'pick-authenticator' action is submitted.
   * @throws {Error} Throws an error if the form submission fails.
   *
   * @example
   * ```typescript
   * // Assuming 'sdk' is an instance of MfaWebAuthnPlatformChallenge
   * try {
   *   await sdk.tryAnotherMethod();
   *   // On success, Auth0 handles redirection to MFA selection.
   * } catch (error) {
   *   console.error("Failed to switch MFA method:", error);
   * }
   * ```
   */
  tryAnotherMethod(options?: TryAnotherMethodOptions): Promise<void>;
}

<ParamField body='reportBrowserError' type='Promise<void>'>

Signale une erreur précise de l’API WebAuthn (provenant de `navigator.credentials.get()`) à Auth0.
Cette méthode doit être utilisée lorsque `verify()` (ou un appel manuel à `navigator.credentials.get()`) échoue en raison
d’un problème WebAuthn côté navigateur (p. ex., annulation par l’utilisateur `NotAllowedError`, expiration du délai).
Elle envoie les détails de l’erreur avec `action: "showError::{errorDetailsJsonString}"` et une `response` vide.

Une promesse résolue lorsque le rapport d’erreur est soumis.

<div id="throws">
  #### Lève
</div>

Lève une erreur si l’envoi du formulaire échoue (p. ex., erreur réseau, état invalide).

```typescript Example
// En supposant que 'sdk' soit une instance de MfaWebAuthnPlatformChallenge
// et que webAuthnDomException soit un objet d’erreur provenant d’un appel navigator.credentials.get() ayant échoué.
try {
  await sdk.reportBrowserError({
    error: { name: webAuthnDomException.name, message: webAuthnDomException.message }
  });
  // Auth0 traitera cette erreur et pourra réafficher la page ou effectuer une redirection.
} catch (submitError) {
  console.error("Échec du signalement de l’erreur WebAuthn du navigateur :", submitError);
}
```

Permet à l’utilisateur de quitter le défi WebAuthn de la plateforme et de sélectionner une autre méthode d’authentification multifacteur (AMF (MFA)).

<ParamField body='verify' type='Promise<void>'>

Lance le challenge de l’authentificateur de plateforme WebAuthn.
Cette méthode appelle en interne `navigator.credentials.get()` en utilisant les options de challenge
fournies dans `screen.publicKey`.
En cas de réussite, elle envoie les informations d’identification obtenues à Auth0 avec `action: "default"`.

Si `navigator.credentials.get()` échoue (p. ex., annulation par l’utilisateur, délai d’attente ou autres erreurs de l’API WebAuthn),
il est recommandé d’appeler `reportBrowserError` avec les détails de l’erreur.

Une promesse résolue lorsque la tentative de vérification est soumise.
                         Une opération réussie entraîne généralement une redirection.

<div id="throws">
  #### Lève
</div>

Lève une erreur si `screen.publicKey` est absent,
                si `navigator.credentials.get()` échoue avec une erreur inattendue qui n’est pas gérée par `getPasskeyCredentials`,
                ou si l’envoi du formulaire à Auth0 échoue.

```typescript Example
// En supposant que 'sdk' soit une instance de MfaWebAuthnPlatformChallenge
try {
  const remember = sdk.screen.showRememberDevice && userCheckedRememberDeviceBox;
  await sdk.verify({ rememberDevice: remember });
  // En cas de réussite, Auth0 gère la redirection.
} catch (error) {
  console.error("Platform authenticator verification failed:", error);
  // S'il s'agit d'une erreur de l’API WebAuthn, signalez-la
  if (error.name && error.message) { // Vérification de base pour une erreur de type DOMException
    await sdk.reportBrowserError({ error: { name: error.name, message: error.message } });
  }
  // Vérifiez sdk.transaction.errors pour les messages de validation côté serveur si la page se recharge.
}
```
