<ParamField body='continueWithNewNickname' type='Promise<void>'>

Soumet le nouveau surnom pour la clé de sécurité WebAuthn.
Cette action correspond au fait que l'utilisateur entre un nouveau surnom et clique sur un bouton « Enregistrer » ou « Continuer ».
Le SDK enverra ce nouveau surnom par requête POST au point de terminaison Auth0 `/u/mfa-webauthn-change-key-nickname`.

Si le nouveau surnom est valide et que la mise à jour réussit, Auth0 redirigera généralement l'utilisateur
vers l'écran suivant approprié (p. ex., retour à la gestion des facteurs d'AMF (MFA) ou à une confirmation de réussite).
Si le surnom n'est pas valide (p. ex., vide, trop long, trop court), Auth0 rendra de nouveau l'écran
« mfa-webauthn-change-key-nickname » et le tableau `transaction.errors` dans le contexte du SDK
sera mis à jour avec les détails de l'échec de validation.

Une promesse qui se résout lorsque l'envoi du formulaire est lancé.
                         Elle ne retourne pas de données directement lors de la résolution, car une redirection ou
                         un nouveau rendu de la page est le résultat le plus fréquent.

<div id="throws">
  #### Lève
</div>

Lève une erreur si `payload.nickname` n'est pas fourni ou n'est pas une chaîne de caractères,
                ou si le `FormHandler` rencontre un problème irrécupérable pendant l'envoi (p. ex., erreur réseau).
                Les erreurs de validation provenant d'Auth0 (comme un surnom invalide) ne sont pas levées comme des erreurs JavaScript,
                mais sont reflétées dans `this.transaction.errors` après l'opération.

```typescript Example
// En supposant que 'sdk' soit une instance de MfaWebAuthnChangeKeyNickname
const newNickname = "My Favorite YubiKey"; // Valeur provenant de la saisie de l'utilisateur
try {
  await sdk.continueWithNewNickname({ nickname: newNickname });
  // En cas de réussite, la page est redirigée.
} catch (error) {
  // Ce bloc catch sert pour les erreurs inattendues lors de l'appel au SDK lui-même.
  console.error("Échec de l'envoi du nouveau surnom :", error);
}
// Après l'await, vérifiez toujours sdk.transaction.errors pour les problèmes de validation côté serveur.
if (sdk.transaction.errors && sdk.transaction.errors.length > 0) {
  sdk.transaction.errors.forEach(err => {
    if (err.field === 'nickname') {
      // Affichez err.message liée au champ de saisie du surnom.
      // p. ex., "Name is too short", "Name is required"
    }
  });
}
```
