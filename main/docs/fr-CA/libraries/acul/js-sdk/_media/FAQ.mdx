<div id="where-can-i-see-errors-related-to-my-screen-and-how-to-handle-them">
  ## Où puis-je voir les erreurs liées à mon écran et comment les gérer
</div>

La trousse de développement logiciel (SDK) fournit plusieurs façons de détecter et de consulter les erreurs au cours du processus :

1. **Méthode `getErrors` pour les erreurs propres à l'écran**  
   Une fonction accesseur de haut niveau, `getErrors`, peut être utilisée pour récupérer des erreurs d'authentification détaillées lorsque la validation du backend échoue.
   ```javascript
      import Login from "@auth0/auth0-acul-js/login";

      const loginIdManager = new Login();

      const errors = loginIdManager.getErrors();
      if (errors) {
        console.error("Login failed:", errors);
      }
   ```
1. **Erreurs propres à l'écran**  
   Chaque écran inclut une propriété `transaction.errors`, qui renvoie une liste d'erreurs propres à cet écran. Ces erreurs sont générées après la validation finale de la requête, ce qui vous aide à identifier les problèmes qui nécessitent une attention particulière.

En utilisant ces méthodes, vous pouvez facilement suivre et résoudre les erreurs tout au long du flux du SDK.

<div id="can-i-use-this-sdk-from-a-cdn">
  ## Puis-je utiliser cette trousse de développement logiciel (SDK) à partir d’un CDN ?
</div>

À l’heure actuelle, ce SDK est uniquement disponible pour une installation via le registre npm et ne peut pas être utilisé directement à partir d’un CDN.

Pour installer le SDK, exécutez la commande suivante :

```sh
npm install @auth0/auth0-acul-js
```

Une fois que vous l’avez installé, vous pouvez l’importer et l’utiliser dans vos projets JavaScript, TypeScript ou ReactJS.


<div id="can-i-build-a-reactjs-application-with-this-sdk">
  ## Puis‑je créer une application ReactJS avec cette trousse de développement logiciel (SDK)?
</div>

Oui, cette trousse de développement logiciel (SDK) peut être facilement intégrée à une application ReactJS.

Bien que la trousse de développement logiciel (SDK) soit écrite en JavaScript « vanilla », elle est entièrement compatible avec tous les frameworks JavaScript, y compris React, Angular, Vue et autres. Vous pouvez utiliser le SDK de manière transparente dans vos composants ou Hooks React.
