    <Section id={sections[0].id} title={sections[0].title} stepNumber="1" isSingleColumn>
      Pour utiliser les services Auth0, vous devez avoir une application configurée dans Auth0 Dashboard. C’est dans l’application Auth0
      que vous configurerez le fonctionnement de l’authentification pour votre projet.

      ### Configurer une application Auth0

      Utilisez le sélecteur interactif pour créer une nouvelle application Auth0 ou sélectionner une application Auth0
      **Native** existante. Chaque application dans Auth0 se voit attribuer un ID client alphanumérique unique que le code de votre application
      utilisera pour appeler les API Auth0 par l’entremise de la trousse de développement logiciel (SDK).

      Tous les paramètres que vous configurez à l’aide de ce démarrage rapide seront automatiquement mis à jour pour votre application dans le [Dashboard](https://manage.auth0.com/dashboard/us/auth0-dsepaid/), où vous pourrez gérer vos applications par la suite.

      Si vous préférez explorer une configuration complète, vous pouvez plutôt consulter une application d’exemple.

      ### Configurer les URL de rappel et de déconnexion

      Les URL de rappel et de déconnexion sont les URL qu’Auth0 appelle pour rediriger vers votre application. Auth0 appelle l’URL de
      rappel après avoir authentifié l’utilisateur, et l’URL de déconnexion après avoir supprimé le témoin de session. Si les URL de rappel
      et de déconnexion ne sont pas définies, les utilisateurs ne pourront pas ouvrir ni fermer une session dans l’application et recevront un message d’erreur.

      Définissez les URL de rappel et de déconnexion sur les valeurs suivantes, selon votre plateforme.

      <Info>
        Sur Android, la valeur de l’espace réservé `SCHEME` peut être `https` ou un autre
        schéma personnalisé. Les schémas `https` exigent l’activation de [Android App Links](https://auth0.com/docs/get-started/applications/enable-android-app-links-support).

        Sur iOS 17.4+ et macOS 14.4+, il est possible d’utiliser des Universal Links (schéma `https`) comme
        URL de rappel et de déconnexion. Lorsque cette option est activée, le SDK utilisera par défaut un schéma d’URL personnalisé sur les versions
        iOS / macOS plus anciennes — l’[identifiant de bundle](https://developer.apple.com/documentation/appstoreconnectapi/bundle_ids) de votre application. **Cette fonctionnalité nécessite Xcode 15.3+ et un compte Apple
        Developer payant**.
      </Info>

      #### Android

      ```
      SCHEME://{yourDomain}/android/YOUR_PACKAGE_NAME/callback
      ```

      #### iOS

      ```
      https://{yourDomain}/ios/YOUR_BUNDLE_IDENTIFIER/callback,
      ```

      ```
      YOUR_BUNDLE_IDENTIFIER://{yourDomain}/ios/YOUR_BUNDLE_IDENTIFIER/callback
      ```

      #### macOS

      ```
      https://{yourDomain}/macos/YOUR_BUNDLE_IDENTIFIER/callback,
      ```

      ```
      YOUR_BUNDLE_IDENTIFIER://{yourDomain}/macos/YOUR_BUNDLE_IDENTIFIER/callback
      ```

      Par exemple, si l’identifiant de bundle iOS était `com.example.MyApp` et que votre domaine Auth0 était
      `example.us.auth0.com`, cette valeur serait alors la suivante :

      ```
      https://example.us.auth0.com/ios/com.example.MyApp/callback,
      ```

      ```
      com.example.MyApp://example.us.auth0.com/ios/com.example.MyApp/callback
      ```

      Si vous ne développez pas pour la plateforme Android, passez cette étape.

      La Trousse de développement logiciel (SDK) nécessite des espaces réservés (placeholders) dans le manifeste. Auth0 utilise des espaces réservés en interne pour définir un
      `intent-filter`, qui intercepte l’URL de rappel d’authentification. Vous devez définir le domaine du tenant Auth0
      et le schéma de l’URL de rappel.

      L’[exemple](https://github.com/auth0-samples/auth0-flutter-samples/tree/main/sample/android) utilise les espaces réservés suivants :

      * `auth0Domain` : le domaine de votre tenant Auth0. En général, vous le trouverez dans l’Auth0 Dashboard,
        sous **Application Settings**, dans le champ **Domain**. Si vous utilisez un Domaine personnalisé, vous
        devez plutôt définir cette valeur sur celle de votre Domaine personnalisé.
      * `auth0Scheme` : le schéma à utiliser. Il peut s’agir d’un schéma personnalisé ou de https si vous voulez utiliser [Android App Links](https://auth0.com/docs/applications/enable-android-app-links).
        Vous pouvez en apprendre davantage sur la configuration de cette valeur dans le fichier [Auth0.Android SDK README](https://github.com/auth0/Auth0.Android#a-note-about-app-deep-linking).

      <Info>
        Vous n’avez pas besoin de déclarer un `intent-filter` spécifique pour votre activité, car vous avez défini
        les espaces réservés du manifeste avec vos valeurs **Domain** et **Scheme** Auth0. La bibliothèque gère
        la redirection pour vous.
      </Info>

      Exécutez **Sync Project with Gradle Files** dans Android Studio pour appliquer vos modifications.

      ### Configurer le Team ID et l’identifiant de bundle

      Accédez à la [page de paramètres](https://manage.auth0.com/#/applications/\{yourClientId}/settings) de votre application Auth0, faites défiler jusqu’au bas de la page, puis ouvrez **Advanced
      Settings &gt; Device Settings**. Dans la section **iOS**, définissez **Team ID** sur votre [Apple Team ID](https://developer.apple.com/help/account/manage-your-team/locate-your-team-id/), et **App ID** sur l’identifiant de bundle de votre application.

      <Frame>
        <img src="/docs/images/cdy7uua7fh8z/7wetuICumueyqt6dbB32ro/df8658dff11b699ac94c578ff3dd2d8a/IOS_Settings_-_English.png" />
      </Frame>

      Cette opération ajoutera votre application au fichier `apple-app-site-association` de votre tenant Auth0.

      ### Ajouter la capacité de domaine associé

      Ouvrez votre application dans Xcode en exécutant `open ios/Runner.xcworkspace` (ou
      `open macos/Runner.xcworkspace` pour macOS). Accédez à l’[onglet **Signing and Capabilities**](https://developer.apple.com/documentation/xcode/adding-capabilities-to-your-app#Add-a-capability) des paramètres de la cible **Runner**, puis cliquez sur le bouton **+
      Capability**. Sélectionnez ensuite **Associated Domains**.

      <Frame>
        <img src="/docs/images/cdy7uua7fh8z/3GO76kXynaieKs5CSj3UTp/1cc577b56d00bc3bad877e31b848c1ec/ios-xcode-capabilities.png" />
      </Frame>

      Ajoutez ensuite l’[entrée](https://developer.apple.com/documentation/xcode/configuring-an-associated-domain#Define-a-service-and-its-associated-domain) suivante sous **Associated Domains** :

      ```
      webcredentials:{yourDomain}
      ```

      <Info>
        Pour que le domaine associé fonctionne, votre app doit être signée avec le certificat de votre équipe **même lorsqu’elle est compilée pour le simulateur iOS**. Assurez-vous d’utiliser l’équipe Apple dont le Team ID est configuré
        dans la page des paramètres de votre Application Auth0.
      </Info>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5" isSingleColumn>
      Universal Login est le moyen le plus simple de configurer
      l’authentification dans votre application. Nous recommandons de l’utiliser pour obtenir la meilleure expérience, la meilleure sécurité et la gamme la plus complète de
      fonctionnalités.

      Intégrez Auth0 Universal Login dans votre application Flutter en utilisant la classe `Auth0`. Redirigez vos utilisateurs vers
      la page Auth0 Universal Login à l’aide de `webAuthentication().login()`. Il s’agit d’un `Future` et
      vous devez attendre sa résolution pour pouvoir récupérer les jetons de l’utilisateur.

      **Android** : si vous utilisez un schéma personnalisé, transmettez ce schéma à la méthode de connexion afin que la Trousse de développement logiciel (SDK) puisse acheminer
      vers la page de connexion, puis en revenir correctement :

      ```
      await auth0.webAuthentication(scheme: 'VOTRE SCHÉMA PERSONNALISÉ').login();
      ```

      Lorsqu’un utilisateur se connecte, il est redirigé vers votre application. Vous pouvez ensuite accéder aux jetons d’ID et d’accès
      pour cet utilisateur.

      <Note>
        ##### Point de contrôle

        Ajoutez un bouton à votre application qui appelle `webAuthentication().login()` et qui connecte l’utilisateur à votre
        application. Vérifiez que vous êtes bien redirigé vers Auth0 pour l’authentification, puis de retour vers votre application.

        Vérifiez que vous pouvez accéder aux jetons dans le résultat de l’appel à `login`.
      </Note>
