  <Content>
    Auth0 vous permet d&#39;ajouter rapidement l&#39;authentification à presque n&#39;importe quel type d&#39;application. Ce guide explique comment
    intégrer Auth0, ajouter l&#39;authentification et afficher les informations de profil utilisateur dans n&#39;importe quelle application WPF et WinForms en utilisant
    les trousses de développement logiciel (SDK) Auth0 pour [WPF](https://www.nuget.org/packages/Auth0.OidcClient.WPF/) et [WinForms](https://www.nuget.org/packages/Auth0.OidcClient.WinForms).

    Pour utiliser ce démarrage rapide, vous devez :

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1" isSingleColumn>
      Pour utiliser les services Auth0, vous devez disposer d’une Application configurée dans l’Auth0 Dashboard. C’est dans cette Application Auth0 que vous configurerez le fonctionnement de l’authentification pour le projet que vous développez.

      ### Configurer une application

      Utilisez le sélecteur interactif pour créer une nouvelle Application Auth0 ou sélectionner une Application existante qui représente
      le projet que vous voulez intégrer. Chaque Application dans Auth0 se voit attribuer une ID client alphanumérique et unique
      que le code de votre application utilisera pour appeler les API Auth0 au moyen de la trousse de développement logiciel (SDK).

      Tous les paramètres que vous configurez à l’aide de ce démarrage rapide seront automatiquement mis à jour pour votre Application dans le [Dashboard](https://manage.auth0.com/#/), où vous
      pourrez gérer vos Applications par la suite.

      Si vous préférez explorer une configuration complète, vous pouvez plutôt consulter une application d’exemple.

      ### Configurer les URL de rappel (Callback)

      Une URL de rappel est une URL dans votre application vers laquelle vous souhaitez qu’Auth0 redirige les utilisateurs après leur
      authentification. Si elle n’est pas définie, les utilisateurs ne seront pas renvoyés vers votre application après leur connexion.

      <Info>
        Si vous suivez notre projet d’exemple, définissez cette valeur à
        `http://{yourDomain}:4200/mobile`.
      </Info>

      ### Configurer les URL de déconnexion

      Une URL de déconnexion est une URL dans votre application vers laquelle vous souhaitez qu’Auth0 redirige les utilisateurs après leur
      déconnexion. Si elle n’est pas définie, les utilisateurs ne pourront pas se déconnecter à partir de votre application et recevront une erreur.

      <Info>
        Si vous suivez notre projet d’exemple, définissez cette valeur à
        `http://{yourDomain}:4200/mobile`.
      </Info>

    <Section id={sections[1].id} title={sections[1].title} stepNumber="2" isSingleColumn>
      Auth0 fournit une trousse de développement logiciel (SDK) pour [WPF](https://www.nuget.org/packages/Auth0.OidcClient.WPF/) et [WinForms](https://www.nuget.org/packages/Auth0.OidcClient.WinForms) afin de simplifier la mise en œuvre de l’authentification Auth0 dans les applications WPF et WinForms.

      Utilisez le gestionnaire de paquets NuGet (Tools -&gt; Library Package Manager -&gt; Package Manager Console) pour installer le package `Auth0.OidcClient.WPF` ou `Auth0.OidcClient.WinForms`, selon que vous créez une application WPF ou Windows Forms.

      Vous pouvez aussi utiliser la console du gestionnaire de paquets NuGet (`Install-Package`) ou la CLI `dotnet` (`dotnet add`).

      ```
      Install-Package Auth0.OidcClient.WPF
      Install-Package Auth0.OidcClient.WinForms
      ```

      ```
      dotnet add Auth0.OidcClient.WPF
      dotnet add Auth0.OidcClient.WinForms
      ```

    <Section id={sections[2].id} title={sections[2].title} stepNumber="3" isSingleColumn>
      Pour intégrer Auth0 à votre application, instanciez un objet de la classe Auth0Client en lui passant une instance
      d’Auth0ClientOptions qui contient votre domaine Auth0 et votre ID client.

      Par défaut, la trousse de développement logiciel (SDK) utilisera [WebView2](https://learn.microsoft.com/en-us/microsoft-edge/webview2/) pour .NET6 et les versions ultérieures, tout en s’appuyant sur l’ancienne WebView
      pour les applications utilisant une version antérieure à .NET6.

      <Note>
        ##### Point de contrôle

        Votre `Auth0Client` devrait maintenant être correctement instancié. Exécutez votre application pour vérifier que :

        * Le `Auth0Client` est instancié correctement.
        * Votre application ne génère aucune erreur liée à Auth0.
      </Note>

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4" isSingleColumn>
      Maintenant que vous avez configuré votre Application Auth0 et la trousse de développement logiciel (SDK) Auth0, vous devez configurer la connexion pour votre projet.
      Pour ce faire, vous utiliserez la méthode `LoginAsync()` du SDK pour créer un bouton de connexion qui redirige les utilisateurs
      vers la page Auth0 Universal Login. Une fois qu’un utilisateur s’est authentifié avec succès, il sera redirigé vers l’URL de rappel
      que vous avez configurée plus tôt dans ce Démarrage rapide.

      S’il n’y a pas d’erreur, vous pouvez accéder à `User`, `IdentityToken`,
      `AccessToken` et `RefreshToken` à partir de l’objet `LoginResult` retourné par
      `LoginAsync()`.

      <Note>
        ##### Point de contrôle

        Vous devriez maintenant pouvoir ouvrir une session ou vous inscrire à l’aide d’un nom d&#39;utilisateur et d’un mot de passe.

        Cliquez sur le bouton de connexion et vérifiez que :

        * Votre Application WPF ou WinForms vous redirige vers la page Auth0 Universal Login.
        * Vous pouvez ouvrir une session ou vous inscrire.
        * Auth0 vous redirige vers votre application.
      </Note>

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5" isSingleColumn>
      Les utilisateurs qui se connectent à votre projet devront aussi pouvoir se déconnecter. Créez un bouton de déconnexion à l’aide de la Trousse de développement logiciel (SDK) et de la méthode
      `LogoutAsync()`. Lorsque les utilisateurs se déconnectent, ils sont redirigés vers votre point de terminaison de déconnexion Auth0,
      qui les redirige alors immédiatement vers l’URL de déconnexion que vous avez configurée précédemment dans ce démarrage rapide.

      <Note>
        ##### Point de contrôle

        Exécutez votre application et cliquez sur le bouton de déconnexion, puis vérifiez que :

        * Votre application WPF ou WinForms vous redirige vers l’adresse que vous avez spécifiée comme l’une des URL de déconnexion autorisées dans les paramètres de votre Application.
        * Vous n’êtes plus connecté à votre application.
      </Note>

    <Section id={sections[5].id} title={sections[5].title} stepNumber="6" isSingleColumn>
      Maintenant que vos utilisateurs peuvent se connecter et se déconnecter, vous voudrez probablement pouvoir récupérer les informations de profil associées aux utilisateurs authentifiés.
      Par exemple, vous pourriez vouloir afficher le nom d’un utilisateur connecté ou sa photo de profil dans votre projet.

      La Trousse de développement logiciel (SDK) Auth0 pour WPF et WinForms fournit les informations de l’utilisateur au moyen de la propriété `LoginResult.User`.

      <Note>
        ##### Point de contrôle

        Vérifiez que vous pouvez afficher le nom de l’utilisateur ou une autre propriété de l’utilisateur après la connexion.
      </Note>
