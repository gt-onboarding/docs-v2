Vous pouvez configurer ACUL à l&#39;aide de l&#39;Auth0 Dashboard, de la Management API et d&#39;outils d&#39;infrastructure as code (IaC).

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour configurer ACUL, assurez-vous que vos fichiers de configuration personnalisés et vos ressources sont hébergés sur un CDN public.
</Callout>

<div id="auth0-dashboard">
  ## Auth0 Dashboard
</div>

Vous pouvez utiliser le Auth0 Dashboard pour personnaliser vos écrans avec ACUL :

A. Accédez à [Dashboard &gt; Branding &gt; Universal Login](https://manage.auth0.com/dashboard/#/universal-login/customizations-new)

1. Sélectionnez **Customize authentication screens**.
   2. Sélectionnez un <code>screen</code> à personnaliser.
   3. Dans **Settings &gt; Rendering mode**, sélectionnez **Advanced mode**.
   4. (Facultatif) Sélectionnez **Use custom page template**.
   5. (Facultatif) Sélectionnez **Advanced mode filters**.

<Frame>
  <img src="/docs/images/cdy7uua7fh8z/ACUL/aculDashboard.png" alt="Page ACUL dans Auth0 Dashboard" />
</Frame>

6. Sélectionnez **Head Tags** pour ajouter vos propres balises HTML `Head`.
   7. (Facultatif) Sélectionnez **Additional Data** pour choisir quelles données supplémentaires envoyer au navigateur.

B. Sélectionnez **Save &amp; Publish**

**Utilisez l&#39;objet de configuration de rendu ACUL**

La configuration de rendu ACUL est un objet JSON unique qui sert à activer ACUL et à répertorier toutes les propriétés disponibles, par écran, qui sont constituées d’éléments spécifiques, de données de contexte et de logique conditionnelle applicable. L&#39;objet de configuration de rendu ACUL peut être utilisé avec des outils d’IaC et l’Auth0 Management API.

Pour en savoir plus, consultez l’endpoint [`/v2/prompts/patch-rendering`](https://auth0.com/docs/api/management/v2/prompts/patch-rendering).

<div id="the-acul-rendering-configuration-object">
  ## L&#39;objet de configuration de rendu ACUL
</div>

La configuration de rendu ACUL est un objet JSON unique utilisé pour activer ACUL et énumérer, pour chaque écran, toutes les propriétés disponibles, qui comprennent des ressources spécifiques, des données de contexte et une logique conditionnelle applicable. L’objet de configuration de rendu ACUL peut être utilisé avec des outils d’IaC et l’Auth0 Management API.

Pour en savoir plus, consultez le point de terminaison [`/v2/prompts/patch-rendering`](https://auth0.com/docs/api/management/v2/prompts/patch-rendering).

<div id="infrastructure-as-code-iac">
  ## Infrastructure-as-Code (IaC)
</div>

Configure ACUL avec des outils IaC, notamment :

* La commande [`auth0 acul`](https://auth0.github.io/auth0-cli/auth0_acul.html) de la **CLI Auth0**.
  * La ressource [`auth0_prompt_screen_renderer`](https://registry.terraform.io/providers/auth0/auth0/latest/docs/resources/prompt_screen_renderer) du **fournisseur Terraform Auth0**.
  * La ressource [`prompts`](https://github.com/auth0/auth0-deploy-cli/blob/master/docs/resource-specific-documentation.md#prompts) de **Auth0 Deploy CLI**.

<div id="auth0-management-api">
  ## Auth0 Management API
</div>

Utilisez les points de terminaison de la Management API propres à ACUL pour configurer les écrans Universal Login individuellement ou pour mettre à jour plusieurs écrans avec un seul appel d’API. Vous devez inclure un [jeton d’accès Management API](/docs/fr-CA/secure/tokens/access-tokens/management-api-access-tokens) dans la requête.

**Configuration individuelle d’écran**

* Le [point de terminaison `GET /v2/prompts/{prompt}/screen/{screen}/rendering`](/docs/fr-CA/api/management/v2/prompts/get-rendering) vous permet de récupérer la configuration d’un seul écran.

  * Le [point de terminaison `PATCH /v2/prompts/{prompt}/screen/{screen}/rendering`](/docs/fr-CA/api/management/v2/prompts/patch-rendering) vous permet de configurer ACUL sur un seul écran.

**Configuration groupée des écrans**

* Le [point de terminaison `GET /v2/prompts/rendering`](/docs/fr-CA/api/management/v2/prompts/get-all-rendering) vous permet de récupérer les configurations de tous les écrans.

  * Le [point de terminaison `PATCH /v2/prompts/rendering`](/docs/fr-CA/api/management/v2/prompts/patch-bulk-rendering) vous permet de mettre à jour plusieurs écrans en bloc.
