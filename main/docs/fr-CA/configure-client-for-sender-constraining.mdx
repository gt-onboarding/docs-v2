<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Pour utiliser les fonctionnalités Highly Regulated Identity, vous devez avoir un forfait Enterprise avec le module complémentaire Highly Regulated Identity. Consultez la page [Auth0 Pricing](https://auth0.com/pricing/) pour plus de détails.
</Callout>

Lorsque vous exigez Sender Constraining pour une application cliente, les jetons d’accès sont restreints à cette application. Auth0 vérifie les requêtes pour s’assurer que seule l’application qui a demandé le jeton peut l’utiliser pour accéder à la ressource associée. Pour en savoir plus, consultez [Configure Sender Constraining](/docs/fr-CA/secure/sender-constraining/configure-sender-constraining).

<Tabs>
  <Tab title="Auth0 Dashboard">
    1. Accédez à [Dashboard &gt; Applications &gt; Applications](https://manage.auth0.com/#/applications). Sélectionnez l’application que vous voulez configurer.
    2. Sous **Settings**, faites défiler jusqu’à **Token Sender-Constraining**.
    3. Activez **Require Sender Constraining**. Désactivez pour retirer l’exigence de Sender Constraining pour l’application.

    <Frame>![](/docs/images/cdy7uua7fh8z/3BoGW9NEYKPBkfk2tPBL7l/0f884ac3f64f17b23fd6a0e9436bb472/Screenshot_2024-07-23_at_1.30.19_PM.png)</Frame>
  </Tab>

  <Tab title="Management API">
    Pour configurer un client pour Sender Constraining, utilisez la [Management API](https://auth0.com/docs/api/management/v2).

    Pour exiger Sender Constraining pour un client, envoyez une requête PATCH afin de [mettre à jour les paramètres du client](https://auth0.com/docs/api/management/v2/clients/patch-clients-by-id). Réglez le paramètre `require_proof_of_possession` sur `true`.

    ```json
    "require_proof_of_possession": true
    ```

    Pour retirer l’exigence de Sender Constraining, réglez le paramètre `require_proof_of_possession` sur `false`.

    ```json
    "require_proof_of_possession": false
    ```
  </Tab>
</Tabs>
