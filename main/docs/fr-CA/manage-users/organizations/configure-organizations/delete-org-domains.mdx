<Warning>
  Les domaines de découverte d’organisation sont actuellement offerts en [accès anticipé](/docs/fr-CA/troubleshoot/product-lifecycle/product-release-stages). En utilisant cette fonctionnalité, vous acceptez les conditions applicables de l’essai gratuit figurant dans le [contrat-cadre d’abonnement](https://www.okta.com/legal/) d’Okta.
</Warning>

Lorsque vous supprimez un domaine de découverte d’organisation, l’association entre le domaine et l’organisation est définitivement supprimée.

<Warning>
  Une fois que vous avez confirmé la suppression d’un domaine de découverte, cette opération ne peut pas être annulée.
</Warning>

Supprimez un domaine d’organisation dans Auth0 Dashboard ou avec la Management API.

<Tabs>
  <Tab title="Auth0 Dashboard">
    Pour supprimer un domaine d’organisation dans Auth0 Dashboard :

    1. Accédez à [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations).
    2. Sélectionnez l’onglet **Domains**.
    3. Choisissez **Add Domain**.
    4. Repérez le domaine que vous voulez supprimer.
    5. Ouvrez le menu des actions (⋮) à côté du domaine et sélectionnez **Delete**.
    6. Confirmez la suppression.
  </Tab>

  <Tab title="Management API">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      <b>Trouvez votre domaine Auth0</b>

      Votre domaine Auth0 est le nom de votre tenant, votre sous-domaine régional et `auth0.com`, sauf si votre tenant a été créé avant juin 2020 dans la région États-Unis.

      Exemple : si le nom de votre tenant est Travel0 et que votre région est US, alors votre domaine Auth0 est `travel0.us.auth0.com.` Si votre tenant a été créé avant juin 2020 dans la région États-Unis, alors votre nom de domaine serait `https://travel0.auth0.com`.

      Si vous avez des domaines personnalisés configurés et vérifiés dans Auth0 Dashboard, votre domaine est votre nom de domaine personnalisé. Pour en savoir plus, consultez [Custom Domains](/docs/fr-CA/customize/custom-domains).
    </Callout>

    Appelez le point de terminaison `Delete Organization Discovery Domain` de la Management API. Assurez-vous de remplacer les valeurs fictives par les valeurs appropriées provenant de votre tenant.

    ```bash
    curl --request DELETE \
      --url 'https://{yourDomain}/api/v2/organizations/{ORGANIZATION_ID}/discovery-domains/{ORGANIZATION_DOMAIN_ID}' \
      --header 'authorization: Bearer {MGMT_API_ACCESS_TOKEN}'
    ```

    | <b>Valeur</b> | <b>Description</b> |
    | ----- | ----------- |
    | `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour Management API](/docs/fr-CA/tokens/management-api-access-tokens) avec la portée `delete:organization`. |
    | `ORGANIZATION_ID` | ID de l’organisation à laquelle vous voulez ajouter le domaine de découverte (par exemple, `org_abc123`). |
    | `ORGANIZATION_DOMAIN_ID` | Obligatoire. Domaine de courriel que vous voulez associer à Organization Discovery (par exemple, `example.com`).|

    | <b> Code d’état </b>| <b> Description de l’erreur </b>| <b>Message</b> |<b>Cause</b>|
    | ------- | ------- | ------- | ------- |
    | `204` |  | Le domaine de découverte a été supprimé. ||
    | `400` | `invalid_uri` | URI de requête non valide. Le message varie selon la cause. | Le chemin n’est pas valide. |
    | `401` | | Jeton non valide. ||
    | `401` | | Signature non valide reçue pour la validation du JSON Web Token (JWT). ||
    | `403` | `insufficient_scope` | Portée insuffisante ; l’une des portées suivantes était attendue : `delete:organization_discovery_domains`. | Tentative de lecture ou d’écriture d’un champ qui n’est pas permis avec les scopes du jeton Bearer fourni. |
    | `404` | | Le domaine d’organisation n’existe pas. | |
    | `429` |  | Trop de requêtes. Vérifiez les en-têtes `X-RateLimit-Limit`, `X-RateLimit-Remaining` et `X-RateLimit-Reset`. ||
  </Tab>
</Tabs>
