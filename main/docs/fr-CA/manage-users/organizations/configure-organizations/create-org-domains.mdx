Utilisez les domaines d’organisation lors des flux de connexion pour afficher une invite permettant aux utilisateurs de choisir la bonne organisation. Une fois configurés, les domaines vérifiés aident Auth0 à détecter automatiquement l’organisation d’un utilisateur ou à restreindre les options d’organisation lorsque les utilisateurs saisissent leur adresse de courriel.

Pour en savoir plus, consultez [Flux de connexion pour les organisations](/docs/fr-CA/manage-users/organizations/login-flows-for-organizations).

<div id="create-an-organization-domain">
  ## Créer un domaine d&#39;organisation
</div>

Créez des domaines d&#39;organisation dans Auth0 Dashboard ou avec Management API.

<Tabs>
  <Tab title="Auth0 Dashboard">
    Pour créer un domaine d&#39;organisation dans Auth0 Dashboard :

    1. Accédez à [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations).
    2. Sélectionnez l’onglet **Domains**.
    3. Choisissez **Add Domain**.
    4. Saisissez les informations de domaine et l’état.
    5. Sélectionnez **Add Domain**.

    | Champ | Description |
    | ----- | ----------- |
    | Domaine | Domaine de courriel à associer pour la découverte d’organisation (par exemple, `Travel0.com`). Doit respecter un format de domaine valide. L’état est à « en attente » par défaut jusqu’à la confirmation de la propriété. |
  </Tab>

  <Tab title="Management API">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      <b>Trouver votre domaine Auth0</b>

      Votre domaine Auth0 est le nom de votre tenant, votre sous-domaine régional et `auth0.com`, sauf si votre tenant a été créé avant juin 2020 dans la région US.

      Exemple : si le nom de votre tenant est Travel0 et que votre région est US, alors votre domaine Auth0 est `travel0.us.auth0.com.` Si votre tenant a été créé avant juin 2020 dans la région US, alors le nom de votre domaine serait `https://travel0.auth0.com`.

      Si vous avez des domaines personnalisés configurés et vérifiés dans Auth0 Dashboard, votre domaine est votre domaine personnalisé. Pour en savoir plus, consultez [Custom Domains](/docs/fr-CA/customize/custom-domains).
    </Callout>

    Appelez le point de terminaison `Create Organization Discovery Domain` de Management API. Assurez-vous de remplacer les valeurs d’espace réservé par les valeurs appropriées de votre tenant.

    ```bash
    curl --request POST \
      --url 'https://{yourDomain}/api/v2/organizations/{ORGANIZATION_ID}/discovery-domains' \
      --header 'authorization: Bearer {MGMT_API_ACCESS_TOKEN}' \
      --header 'cache-control: no-cache' \
      --header 'content-type: application/json' \
      --data '{
        "domain": "{DOMAIN}",
        "status": "{STATUS}"
      }'
    ```

    | Espace réservé | Description |
    | ----- | ----------- |
    | `MGMT_API_ACCESS_TOKEN` | [Jeton d’accès pour Management API](/docs/fr-CA/tokens/management-api-access-tokens) avec la portée `create:organization_discovery_domains`. |
    | `ORGANIZATION_ID` | ID de l’organisation à laquelle vous voulez ajouter le domaine de découverte (par exemple, `org_abc123`). |
    | `DOMAIN` | Obligatoire. Domaine de courriel que vous voulez associer pour la découverte d’organisation (par exemple, example.com). |
    | `STATUS` | Optionnel. État initial du domaine. Valeurs courantes : `pending`, `verified`. |

    | Code d’état| Description de l’erreur | Message | Cause |
    | --- | --- | --- | --- |
    | `201` |  | Domaine de découverte créé avec succès. L’enregistrement de domaine a été ajouté à l’organisation. ||
    | `400` | `invalid_body` | Corps de requête non valide. Le message varie selon la cause. | La charge utile de la requête n’est pas valide. |
    | `400` | `invalid_query_string` | Chaîne de requête non valide. Le message varie selon la cause. | La chaîne de requête n’est pas valide. |
    | `401` | | Jeton non valide. ||
    | `403` | `insufficient_scope` | Portée insuffisante; l’une des portées suivantes est requise : `create:organization_discovery_domains`. | Tentative d’écriture dans un champ qui n’est pas autorisé avec les scopes du jeton Bearer fourni. |
    | `409` | `conflict` | Un domaine d’organisation portant le même nom existe déjà. | Un domaine d’organisation portant le même nom existe déjà. |
    | `429` | | Trop de requêtes. Vérifiez les en-têtes `X-RateLimit-Limit`, `X-RateLimit-Remaining` et `X-RateLimit-Reset`. ||
  </Tab>
</Tabs>
