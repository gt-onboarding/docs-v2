Dashboard ユーザーは、不要になった、または利用できなくなった <Tooltip tip="Multi-factor authentication（MFA、多要素認証）: ユーザー名とパスワードに加えて、SMS で送信されるコードなどの追加要素を使用するユーザー認証プロセス。" cta="用語集を表示" href="/docs/ja-JP/glossary?term=multi-factor+authentication">multi-factor authentication</Tooltip>（MFA）要素を削除または変更できます。Dashboard ユーザー向けの MFA について詳しくは、「[Multi-Factor Authentication for Dashboard Users](/docs/ja-JP/get-started/manage-dashboard-access/add-change-remove-mfa)」を参照してください。

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  Auth0 では、最も安全かつ使いやすい認証手段として WebAuthn 要素を推奨しています。詳細については、「[FIDO Authentication with WebAuthn](/docs/ja-JP/secure/multi-factor-authentication/fido-authentication-with-webauthn)」を参照してください。
</Callout>

<Warning>
  管理者は、MFA を使用するために少なくとも 1 つの要素を有効にする必要があります。Auth0 では、プライマリ デバイスを紛失した場合でもアカウントにアクセスできるよう、複数の要素を設定することを強く推奨します。

  理想的な構成は、次の 3 つの要素を使用することです。

  1. WebAuthn、Guardian、または OTP をプライマリの方法として使用する。
  2. バックアップとして、1 つ以上の SMS 番号を設定する（サブスクリプションプランに紐づくテナントでのみ利用可能）。
  3. 復旧コードを用意する。

  MFA トークンを提示できず、適切なバックアップ手段もない場合、アカウントを復旧できない可能性があります。
</Warning>

<div id="remove-or-change-an-mfa-factor-from-the-dashboard">
  ## Dashboard から MFA 要素を削除または変更する
</div>

<Callout icon="file-lines" color="#0EA5E9" iconType="regular">
  デバイスの生体認証要素（MacBook の Touch Bar、Windows Hello、iOS の Touch ID や Face ID、Android の指紋認証や顔認証など）は、Dashboard から追加することはできません。デバイスの生体認証要素を追加する方法については、「[Dashboard ユーザー向けに Multi-factor Authentication（MFA）を追加する](/docs/ja-JP/get-started/manage-dashboard-access/add-change-remove-mfa/add-mfa)」を参照してください。
</Callout>

現在の MFA 要素を使ってログインできる Dashboard ユーザーは、次の手順に従うことができます。

1. Dashboard の右上隅にある自分のユーザー名をクリックし、[Your profile](https://manage.auth0.com/#/profile) をクリックします。

   <Frame>![Dashboard - Profile - Multi-Factor - Authentication](/docs/images/cdy7uua7fh8z/2NyLNvP92RruwZ46dLPm5x/8ece3df2b550af8dd78368ca36d8a661/MFA.png)</Frame>
2. 使用したい新しい認証要素を探し、その行で **+ ADD** をクリックします。画面の指示に従って登録を完了します。
3. 引き続き [Your profile](https://manage.auth0.com/#/profile) の画面で、今後使用したくない認証要素を探し、**REMOVE** をクリックします。
4. 削除を確認するために **Yes** をクリックします。
5. Auth0 から、現在使用している（古い）MFA 要素で認証するよう求められます。認証が成功すると、Auth0 はその要素を削除します。

<div id="remove-or-change-a-lost-mfa-factor">
  ## 紛失した MFA 要素を削除または変更する
</div>

現在の MFA 要素を使ってログインできないダッシュボードユーザーは、次の手順に従ってください。

1. ダッシュボードへのログインを試します。Auth0 により、現在設定されている認証要素での認証を求められます。
2. Auth0 に紛失したデバイスまたは認証情報での認証を求められたら、**Try another method** をクリックします。
3. **Other Methods** ボックスで、別の認証方法をクリックします。
4. ログインしてダッシュボードにアクセスします。
5. ダッシュボード右上隅で自分のユーザー名をクリックし、[Your profile](https://manage.auth0.com/#/profile) をクリックします。

   <Frame>![Dashboard - Profile - Multi-Factor - Authentication](/docs/images/cdy7uua7fh8z/2NyLNvP92RruwZ46dLPm5x/8ece3df2b550af8dd78368ca36d8a661/MFA.png)</Frame>
6. 使えなくなった認証方法を見つけて、**REMOVE** をクリックします。
7. 削除を確定するために **Yes** をクリックします。
8. Auth0 により、再度現在の認証要素を使って認証するよう求められます。本人確認のために手順 2～4 を繰り返します。Auth0 は紛失した要素を削除します。

<div id="get-help-from-auth0-support">
  ## Auth0サポートに問い合わせる
</div>

ロックアウトされ、有効になっているMFA要素のいずれにもアクセスできない場合は、アカウントに再度アクセスできる保証はありません。別の管理者が、あなたに代わってAuth0サポートチケットを提出する必要があります。場合によっては、Auth0が要求を検証し、MFAのリセットを実行できます。ただし、アカウントの所有権を確認できない場合があります。そうした理由から、複数かつ多様な要素を有効にしておくことが重要です。

Auth0のサポートは**エンドユーザー**のアカウントをリセットしません。お客様のアプリケーションおよびAPIにアクセスするアカウントについては、お客様が責任を負うものとします。エンドユーザーアカウントについては、「[ユーザーの管理](/docs/ja-JP/manage-users)」をお読みください。
