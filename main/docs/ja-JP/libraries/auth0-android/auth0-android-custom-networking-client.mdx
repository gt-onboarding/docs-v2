このライブラリを使用すると、よくある構成パターン向けにネットワーククライアントの動作をカスタマイズしたり、独自のネットワーククライアント実装を定義して利用したりできます。

Auth0 クラスには `NetworkingClient` を設定でき、リクエスト送信時にそれが使用されます。デフォルトのクライアントについては、カスタムのタイムアウト値、すべてのリクエストで送信する共通ヘッダー、および（本番環境以外でのデバッグ目的に限り）リクエスト／レスポンス情報をログ出力するかどうかを設定できます。さらに高度な構成が必要な場合は、独自の `NetworkingClient` 実装を指定できます。


<div id="timeout-configuration">
  ### タイムアウトの設定
</div>

export const codeExample1 = `val netClient = DefaultClient(
    connectTimeout = 30,
    readTimeout = 30
)

val account = Auth0("{yourClientId}", "{yourDomain}")
account.networkingClient = netClient`;
