      <ParamField body="MfaPollingOptions" type="object">
        MFA プッシュ通知のポーリング処理を管理するための設定オプションです。

        ポーリングセッションを開始する際にこれらのオプションを指定して、
        チャレンジエンドポイントにどの程度の頻度で問い合わせるかや、成功時・失敗時の処理方法を制御します。

        <Expandable title="プロパティ">
          <ParamField body="intervalMs?" type="number">
            連続するポーリングリクエスト間の間隔（**ミリ秒**単位）。

            * 省略した場合、SDK の内部デフォルト間隔（通常は 5000 ms）が使用されます。
            * 応答性とレート制限のバランスが取れる値を選択してください。
          </ParamField>

          <ParamField body="onCompleted()?" type="void">
            MFA プッシュチャレンジが正常に承認され、ポーリングが完了した際に
            実行される任意のコールバックです。

            * 正常終了後に **1 回だけ** 呼び出され、その後ポーリングは自動的に停止します。
            * ログインフローを進める目的（例: `continue()` の呼び出し）に便利です。
          </ParamField>

          <ParamField body="onError()?" type="void">
            ポーリング中にエラーが発生した場合に呼び出される任意のコールバックです。

            [ULError](/docs/ja-JP/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error) オブジェクトを受け取り、次を含みます:

            * `status` — 失敗したリクエストの HTTP ステータスコード

            * `responseText` — 利用可能な場合は生のレスポンスボディ

            * エラーイベントごとに 1 回呼び出されます。エラーのタイプに応じてポーリングが再試行される場合があります。

            * エラーメッセージを表示したり、カスタムの再試行／中止ロジックをトリガーしたりするために使用できます。
