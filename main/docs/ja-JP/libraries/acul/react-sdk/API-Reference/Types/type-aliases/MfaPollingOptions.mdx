---
title: "型エイリアス: MfaPollingOptions"
---

<ParamField body="MfaPollingOptions" type="object">
  MFA のプッシュ通知ポーリング処理を管理するための設定オプションです。

  ポーリングセッションを開始するときにこれらのオプションを指定して、
  チャレンジエンドポイントにどの頻度で問い合わせるか、および
  成功または失敗時の処理方法を制御します。

  <Expandable title="プロパティ">
    <ParamField body="intervalMs?" type="number">
      連続するポーリングリクエスト間の、**ミリ秒**単位の任意指定の間隔です。

      * 省略した場合は、SDK 内部のデフォルト間隔が使用されます（通常は 5000 ms）。
      * 応答性とレート制限のバランスが取れる値を選択してください。

      ```ts Default Value
      実装依存（一般的には 5000 ms）
      ```
    </ParamField>

    <ParamField body="onCompleted()?" type="void">
      MFA プッシュチャレンジが正常に承認され、ポーリングが完了したときに
      実行される任意指定のコールバックです。

      * 正常終了後に **1 回だけ** 呼び出され、その後ポーリングは自動的に停止します。
      * ログインフローを進める処理（例: `continue()` の呼び出し）に便利です。
    </ParamField>

    <ParamField body="onError()?" type="void">
      ポーリング中にエラーが発生した場合に呼び出される任意指定のコールバックです。

      [ULError](/docs/ja-JP/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error) オブジェクトを受け取り、次の情報を含みます:

      * `status` — 失敗したリクエストの HTTP ステータスコード

      * `responseText` — 利用可能な場合は生のレスポンスボディ

      * エラーイベントごとに 1 回呼び出されます。エラーの種類によってはポーリングを再試行する場合があります。

      * エラーメッセージの表示や、カスタムの再試行/中止ロジックをトリガーするために使用します。

      **プロパティ**

      <ParamField body="error" type={<a href="/docs/libraries/acul/react-sdk/API-Reference/Types/interfaces/Error">Error</a>} />
    </ParamField>
  </Expandable>
</ParamField>
