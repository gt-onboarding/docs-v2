  ## メソッド

<ParamField body='continueWithNewNickname' type='Promise<void>'>

WebAuthn セキュリティキーの新しいニックネームを送信します。
このアクションは、ユーザーが新しいニックネームを入力して「保存」または「続行」ボタンをクリックする操作に対応します。
SDK はこの新しいニックネームを Auth0 の `/u/mfa-webauthn-change-key-nickname` エンドポイントに POST します。

新しいニックネームが有効で更新が成功した場合、Auth0 は通常、ユーザーを
次の適切な画面（例: MFA 要素の管理画面や成功確認画面）にリダイレクトします。
ニックネームが無効な場合（例: 空、長すぎる、短すぎる）、Auth0 は
`mfa-webauthn-change-key-nickname` 画面を再レンダリングし、SDK のコンテキスト内にある
`transaction.errors` 配列が検証失敗の詳細で更新されます。

フォーム送信が開始されたときに解決される Promise です。
解決時にデータを直接返すことはなく、リダイレクトや
ページの再レンダリングが一般的な結果となります。

<div id="throws">
  #### スロー
</div>

`payload.nickname` が指定されていない、または文字列ではない場合、
もしくは送信中に `FormHandler` が回復不能な問題に遭遇した場合（例: ネットワークエラー）にエラーをスローします。
Auth0 からの検証エラー（無効なニックネームなど）は JavaScript エラーとしてはスローされず、
操作後に `this.transaction.errors` に反映されます。

```typescript Example
// 'sdk' が MfaWebAuthnChangeKeyNickname のインスタンスであると仮定
const newNickname = "My Favorite YubiKey"; // ユーザー入力値
try {
  await sdk.continueWithNewNickname({ nickname: newNickname });
  // 成功した場合、ページはリダイレクトされます。
} catch (error) {
  // この catch は、SDK 呼び出し自体で発生した予期しないエラー用です。
  console.error("新しいニックネームの送信に失敗しました:", error);
}
// await の後は、常にサーバー側の検証問題について sdk.transaction.errors を確認します。
if (sdk.transaction.errors && sdk.transaction.errors.length > 0) {
  sdk.transaction.errors.forEach(err => {
    if (err.field === 'nickname') {
      // ニックネーム入力フィールドに関連する err.message を表示します。
      // 例: "Name is too short", "Name is required"
    }
  });
}
```
