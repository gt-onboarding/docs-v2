<div id="methods">
  ## メソッド
</div>

* [useAuth0Themes](/docs/ja-JP/libraries/acul/react-sdk/API-Reference/Hooks/useAuth0Themes): ブランディングコンテキストから取得したフラット化済み設定を含む、現在のテーマオプションを取得するための React フック。
* [useCurrentScreen](/docs/ja-JP/libraries/acul/react-sdk/API-Reference/Hooks/useCurrentScreen): 現在の画面コンテキストと state を取得する React フック。
* [useErrors](/docs/ja-JP/libraries/acul/react-sdk/API-Reference/Hooks/useErrors): ACUL (Advanced Customization of Universal Login) でエラーを読み取り、管理するための React フック。すべてのバリデーションエラーとサーバー側エラーを扱います。エラーを次の 3 種類に分類します: - `server` — Auth0 または独自のバックエンドから返されるエラー。 - `client` — クライアント側バリデーションによるエラー (例: 無効なフォーム入力)。 - `developer` — 誤った統合や SDK の誤用によって発生するエラー。
* [useLoginIdentifiers](/docs/ja-JP/libraries/acul/react-sdk/API-Reference/Hooks/useLoginIdentifiers): 認証フローまたはログイン処理で現在使用されている、アクティブな識別子タイプ (例: `'email'`、`'phone'`、`'username'`) の一覧を返します。
* [useMfaPolling](/docs/ja-JP/libraries/acul/react-sdk/API-Reference/Hooks/useMfaPolling): Auth0 Advanced Customization of Universal Login (ACUL) の画面で、MFA（多要素認証）のプッシュポーリング (例: プッシュ通知での承認待ち) を管理する React フック。
* [useResend](/docs/ja-JP/libraries/acul/react-sdk/API-Reference/Hooks/useResend): ACUL の画面で「再送」アクション (例: 認証コードの再送信) を管理する React フック。
* [useSignupIdentifiers](/docs/ja-JP/libraries/acul/react-sdk/API-Reference/Hooks/useSignupIdentifiers): 現在の画面のサインアップ識別子に基づき、有効化されている識別子 (メール、電話番号、またはユーザー名) と、それぞれの `required` ステータスを含む一覧を返します。
* [usePasswordValidation](/docs/ja-JP/libraries/acul/react-sdk/API-Reference/Hooks/usePasswordValidation): 現在の Auth0 パスワードポリシーに対してパスワードを検証し、パスワードが各ルールを満たしているかどうかを示す構造化された結果を返す React フック。
* [useUsernameValidation](/docs/ja-JP/libraries/acul/react-sdk/API-Reference/Hooks/useUsernameValidation): 現在の Auth0 ユーザー名ポリシーに対してユーザー名を検証する React フック。
