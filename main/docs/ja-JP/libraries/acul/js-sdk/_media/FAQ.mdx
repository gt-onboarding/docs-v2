<div id="where-can-i-see-errors-related-to-my-screen-and-how-to-handle-them">
  ## 自分の画面に関連するエラーの確認方法と、その対処方法
</div>

このソフトウェア開発キット (SDK) では、処理中のエラーを検出および確認するための複数の方法を提供しています。

1. **画面固有エラー用の `getErrors` メソッド**  
   トップレベルの getter 関数 `getErrors` を使用して、バックエンドの検証が失敗した際の詳細な認証エラーを取得できます。
   ```javascript
      import Login from "@auth0/auth0-acul-js/login";

      const loginIdManager = new Login();

      const errors = loginIdManager.getErrors();
      if (errors) {
      console.error("Login failed:", errors);
   }
   ```
1. **画面固有のエラー**  
   各画面には `transaction.errors` プロパティが含まれており、その画面に固有のエラーのリストを返します。これらのエラーはリクエストの最終検証後に生成されるため、対応が必要な問題を特定するのに役立ちます。

これらの方法を使用することで、SDK のフロー全体を通してエラーを容易に追跡し、解決できます。

<div id="can-i-use-this-sdk-from-a-cdn">
  ## この SDK を CDN から利用できますか？
</div>

現在、このソフトウェア開発キット (SDK) は npm レジストリからのインストールにのみ対応しており、CDN から直接利用することはできません。

SDK をインストールするには、次のコマンドを実行します。

```sh
npm install @auth0/auth0-acul-js
```

インストールが完了したら、JavaScript、TypeScript、または ReactJS のプロジェクトでインポートして利用できます。


<div id="can-i-build-a-reactjs-application-with-this-sdk">
  ## このソフトウェア開発キット (SDK) で ReactJS アプリケーションを構築できますか？
</div>

はい、このソフトウェア開発キット (SDK) は ReactJS アプリケーションに簡単に統合できます。

この SDK は Vanilla JavaScript で記述されていますが、React、Angular、Vue などを含むすべての JavaScript ベースのフレームワークと完全に互換性があります。React のコンポーネントやフック内で、この SDK をシームレスに利用できます。
