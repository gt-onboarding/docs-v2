  ## コンストラクター

<ParamField body="MfaWebAuthnChangeKeyNickname" type="Constructor">

新しい `MfaWebAuthnChangeKeyNickname` クラスのインスタンスを初期化します。
グローバルな `universal_login_context` から必要なコンテキスト（画面、トランザクション など）を取得し、
`ScreenOverride` を使用して画面固有のプロパティを設定します。

#### 送出される例外

Universal Login コンテキストが利用できない場合、またはコンテキスト内の画面名が
`MfaWebAuthnChangeKeyNickname.screenIdentifier` と一致しない場合にスローされます。

</ParamField>

<div id="methods">
  ## メソッド
</div>

<ParamField body='continueWithNewNickname' type='Promise<void>'>

ユーザーが指定した WebAuthn セキュリティキーの新しいニックネームを送信します。
このメソッドは、新しい `nickname` と必須の `action: "default"` を含むフォームデータを準備し、
`/u/mfa-webauthn-change-key-nickname` エンドポイントに送信します。

フォーム送信が開始されると解決される Promise。
                         通常、送信が成功するとサーバー側でリダイレクトが行われます。
                         ニックネームが無効な場合やその他のエラーが発生した場合は、ページが
                         再レンダリングされ、`this.transaction.errors` に詳細が格納されます。

<div id="throws">
  #### スローされる例外
</div>

`payload.nickname` が存在しないか、文字列ではない場合にスローされます。
                また、送信中に `FormHandler` が回復不能な問題に遭遇した場合
                （例：ネットワークエラー）にもスローされる可能性があります。Auth0 の検証エラー
                （例："nickname-too-long"）は JS のエラーとしてはスローされず、
                実行後に `this.transaction.errors` から参照できるようになります。
