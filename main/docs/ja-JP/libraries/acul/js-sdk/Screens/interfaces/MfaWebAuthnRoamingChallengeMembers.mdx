````ts Example
export interface MfaWebAuthnRoamingChallengeMembers extends BaseMembers {
  /**
   * WebAuthnチャレンジオプションを含む、画面固有のプロパティとデータ。
   * @type {ScreenMembersOnMfaWebAuthnRoamingChallenge}
   */
  screen: ScreenMembersOnMfaWebAuthnRoamingChallenge;

  /**
   * `screen.data.publicKeyChallengeOptions`で提供されるオプションを使用して`navigator.credentials.get()`を呼び出し、WebAuthnチャレンジを開始します。
   * 成功した場合、結果として得られる認証情報を`action: "default"`としてAuth0に送信します。
   * `navigator.credentials.get()`が既知のWebAuthn APIエラー(`NotAllowedError`など)で失敗した場合、
   * このメソッドは内部的に`reportWebAuthnError`を呼び出してAuth0に通知します。
   *
   * @param {VerifySecurityKeyOptions} [options] - 検証プロセスのオプションパラメータ。
   * `rememberDevice`やその他のカスタムオプションなど。
   * @returns {Promise<void>} 検証試行が送信されたときに解決されるPromise。
   *                          成功した操作は通常リダイレクトになるため、データを直接返しません。
   * @throws {Error} `screen.data.publicKeyChallengeOptions`が欠落している場合、
   *                 `navigator.credentials.get()`が予期しないエラーで失敗した場合、
   *                 またはAuth0へのフォーム送信が失敗した場合にエラーをスローします。
   *
   * @example
   * ```typescript
   * // 'sdk'がMfaWebAuthnRoamingChallengeのインスタンスであると仮定
   * try {
   *   await sdk.verify({ rememberDevice: true });
   *   // 成功時、Auth0がリダイレクトを処理します。
   * } catch (error) {
   *   console.error("Security key verification failed:", error);
   *   // ページがリロードされた場合、サーバー側の検証メッセージについてはsdk.transaction.errorsを確認してください。
   * }
   * ```
   */
  verify(options?: VerifySecurityKeyOptions): Promise<void>;

  /**
   * 特定のWebAuthn APIエラーをAuth0に報告します。
   * このメソッドは、`navigator.credentials.get()`が`verify()`で自動的に処理されない方法で失敗した場合、
   * または開発者が別のアクションを試す前にエラーを明示的に報告したい場合に使用します。
   * エラーの詳細を`action: "showError::{errorDetails}"`として送信します。
   *
   * @param {ReportWebAuthnErrorOptions} options - WebAuthn APIからの`name`と`message`を含む`error`オブジェクト、
   * およびその他のカスタムオプションを含みます。
   * @returns {Promise<void>} エラーレポートが送信されたときに解決されるPromise。
   * @throws {Error} フォーム送信が失敗した場合にエラーをスローします。
   *
   * @example
   * ```typescript
   * // 'sdk'がMfaWebAuthnRoamingChallengeのインスタンスであると仮定
   * // webAuthnErrorは失敗したnavigator.credentials.get()呼び出しからのエラーオブジェクト
   * try {
   *   await sdk.reportWebAuthnError({
   *     error: { name: webAuthnError.name, message: webAuthnError.message }
   *   });
   * } catch (submitError) {
   *   console.error("Failed to report WebAuthn error:", submitError);
   * }
   * ```
   */
  reportWebAuthnError(options: ReportWebAuthnErrorOptions): Promise<void>;

  /**
   * ユーザーがWebAuthnチャレンジをオプトアウトし、別のMFA（多要素認証）方式を選択できるようにします。
   * このアクションは`action: "pick-authenticator"`をAuth0に送信し、
   * ユーザーをMFA（多要素認証）要素選択画面に移動させます。
   *
   * @param {TryAnotherMethodOptions} [options] - リクエストのオプションカスタムパラメータ。
   * @returns {Promise<void>} アクションが送信されたときに解決されるPromise。
   * @throws {Error} フォーム送信が失敗した場合にエラーをスローします。
   *
   * @example
   * ```typescript
   * // 'sdk'がMfaWebAuthnRoamingChallengeのインスタンスであると仮定
   * try {
   *   await sdk.tryAnotherMethod();
   *   // 成功時、Auth0がMFA（多要素認証）選択へのリダイレクトを処理します。
   * } catch (error) {
   *   console.error("Failed to switch MFA method:", error);
   * }
   * ```
   */
  tryAnotherMethod(options?: TryAnotherMethodOptions): Promise<void>;
}

<ParamField body='reportWebAuthnError' type='Promise<void>'>

特定の WebAuthn API エラーを Auth0 に報告します。
`navigator.credentials.get()` が `verify()` では自動的に処理されない形で失敗した場合、または開発者が別のアクションを試す前に明示的にエラーを報告したい場合に、このメソッドを使用します。
このメソッドは、`action: "showError::{errorDetails}"` とともにエラーの詳細を送信します。

エラー報告が送信されると解決される Promise です。

```typescript Example

<ParamField body='verify' type='Promise<void>'>

`screen.data.publicKeyChallengeOptions` で指定されたオプションを使用して `navigator.credentials.get()` を呼び出し、WebAuthn チャレンジを開始します。
成功した場合、`action: "default"` とともに得られた認証情報を Auth0 に送信します。
`navigator.credentials.get()` が既知の WebAuthn API エラー（`NotAllowedError` など）で失敗した場合、このメソッドは内部的に `reportWebAuthnError` を呼び出して Auth0 に通知します。

検証試行が送信されると解決される Promise です。
通常はリダイレクトによって成功が処理されるため、データを直接返しません。

<div id="throws">
  #### Throws
</div>

`screen.data.publicKeyChallengeOptions` が存在しない場合、
`navigator.credentials.get()` が想定外のエラーで失敗した場合、
または Auth0 へのフォーム送信が失敗した場合にエラーをスローします。

```typescript Example
// 'sdk' が MfaWebAuthnRoamingChallenge のインスタンスであると仮定します
try {
  await sdk.verify({ rememberDevice: true });
  // 成功時、Auth0 がリダイレクトを処理します。
} catch (error) {
  console.error("Security key verification failed:", error);
  // ページがリロードされた場合は、サーバー側の検証メッセージについて sdk.transaction.errors を確認してください。
}
```
