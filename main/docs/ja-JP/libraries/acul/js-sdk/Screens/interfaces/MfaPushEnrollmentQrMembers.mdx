      <ParamField body="MfaPollingOptions" type="object">
        MFA（多要素認証）のプッシュ通知ポーリング処理を管理するための構成オプションです。

        これらのオプションはポーリングセッションを開始する際に指定し、
        チャレンジエンドポイントへの問い合わせ頻度や、成功時・失敗時の扱いを制御します。

        <Expandable title="プロパティ">
          <ParamField body="intervalMs?" type="number">
            連続するポーリングリクエスト間の、**ミリ秒**単位の任意の間隔。

            * 省略された場合は、SDK の内部デフォルト間隔（通常 5000 ms）が使用されます。
            * 応答性とレート制限のバランスを考慮した値を選択してください。
          </ParamField>

          <ParamField body="onCompleted()?" type="() => void">
            MFA プッシュチャレンジが正常に承認され、ポーリングが完了した際に
            実行される任意のコールバックです。

            * ちょうど **1 回**だけ呼び出され、その後ポーリングは自動的に停止します。
            * ログインフローを進める処理（例: `continue()` の呼び出し）に利用できます。
          </ParamField>

          <ParamField body="onError()?" type="(error) => void">
            ポーリング中にエラーが発生した場合に呼び出される任意のコールバックです。

            次の情報を含む [ULError](/docs/ja-JP/libraries/acul/js-sdk/Screens/interfaces/Error) オブジェクトを受け取ります:

            * `status` — 失敗したリクエストの HTTP ステータスコード

            * `responseText` — 利用可能な場合の生のレスポンスボディ

            * エラーイベントごとに 1 回呼び出されます。ポーリングはエラーの種類に応じて再試行される場合があります。

            * エラーメッセージの表示や、カスタムの再試行／中止ロジックをトリガーするために使用します。
