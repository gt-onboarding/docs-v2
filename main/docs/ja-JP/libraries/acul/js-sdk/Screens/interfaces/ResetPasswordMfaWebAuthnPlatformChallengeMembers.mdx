<ParamField body='continueWithPasskey' type='Promise<void>'>

`screen.publicKey` で指定されたチャレンジオプションを使用して `navigator.credentials.get()` を呼び出し、
WebAuthn プラットフォーム認証器のチャレンジを開始します。
成功すると、結果として得られた認証情報アサーションを `action: "default"` とともに Auth0 に送信します。

検証試行の送信が完了すると解決される Promise です。
正常に処理された場合、通常はリダイレクトが行われます。

#### 例外

`screen.publicKey` が存在しない場合、
`navigator.credentials.get()` が失敗した場合（例: ユーザーによるキャンセル `NotAllowedError`）、
または Auth0 へのフォーム送信が失敗した場合にエラーをスローします。`navigator.credentials.get()` が
`DOMException` で失敗した場合は、そのエラーを捕捉して `reportBrowserError` を呼び出すことを推奨します。

```typescript Example
// 'sdk' が ResetPasswordMfaWebAuthnPlatformChallenge のインスタンスであると仮定
try {
  await sdk.continueWithPasskey({
    rememberDevice: true // ユーザーがチェックボックスをオンにし、かつ sdk.screen.showRememberDevice が true の場合
  });
  // 成功時、Auth0 がリダイレクトを処理します。
} catch (error) {
  console.error("Platform authenticator verification failed:", error);
  // WebAuthn API のエラー (DOMException) の場合は、これを報告します
  if (error instanceof DOMException && error.name && error.message) {
    await sdk.reportBrowserError({ error: { name: error.name, message: error.message } });
  }
  // ページがリロードされた場合は、サーバー側の検証メッセージについて sdk.transaction.errors を確認します。
}
```

<ParamField body='reportBrowserError' type='Promise<void>'>

`navigator.credentials.get()` API 呼び出し中に発生したブラウザー側のエラーを報告します。
これは、ユーザーのキャンセル、タイムアウト、非対応ハードウェアなどの問題について Auth0 に通知するために使用します。
エラーの詳細は `action: "showError::{errorDetailsJsonString}"` とともに送信されます。

エラー報告の送信が完了すると解決される Promise です。

#### 例外

フォーム送信が失敗した場合（例: ネットワークエラー）にスローされます。

```typescript Example
// 'sdk' が ResetPasswordMfaWebAuthnPlatformChallenge のインスタンスであり、
// 'webAuthnError' が失敗した navigator.credentials.get() 呼び出しからの DOMException であると仮定
try {
  await sdk.reportBrowserError({
    error: { name: webAuthnError.name, message: webAuthnError.message }
  });
  // エラーに基づき、Auth0 はページを再レンダリングするか、またはリダイレクトを行う場合があります。
} catch (submitError) {
  console.error("Failed to report WebAuthn browser error:", submitError);
}
```

<ParamField body='tryAnotherMethod' type='Promise<void>'>

ユーザーが WebAuthn プラットフォームチャレンジをスキップし、別の MFA（多要素認証）方法を選択できるようにします。
このアクションは `action: "pick-authenticator"` を Auth0 に送信し、パスワードリセットフロー用の MFA 要素選択画面へ
ユーザーを遷移させます。

`'pick-authenticator'` アクションが送信されると解決される Promise です。
