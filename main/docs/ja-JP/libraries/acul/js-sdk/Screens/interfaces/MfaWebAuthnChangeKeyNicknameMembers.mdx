  ## メソッド

<ParamField body='continueWithNewNickname' type='Promise<void>'>

WebAuthn セキュリティキーの新しいニックネームを送信します。
このアクションは、ユーザーが新しいニックネームを入力し、「保存」または「続行」ボタンをクリックする操作に対応します。
SDK は、この新しいニックネームを Auth0 の `/u/mfa-webauthn-change-key-nickname` エンドポイントに対して POST します。

新しいニックネームが有効で更新が成功すると、Auth0 は通常、
ユーザーを次の適切な画面（例: MFA 要素の管理画面や成功確認画面など）にリダイレクトします。
ニックネームが無効（例: 空、長すぎる、短すぎる）な場合、Auth0 は
`mfa-webauthn-change-key-nickname` 画面を再描画し、SDK のコンテキスト内にある
`transaction.errors` 配列が、検証失敗の詳細で更新されます。

フォーム送信の開始時に解決される Promise です。
解決時にデータを直接返すことはなく、リダイレクトまたは
ページの再描画が一般的な結果となります。

<div id="throws">
  #### Throws
</div>

`payload.nickname` が指定されていない、または文字列ではない場合、
あるいは送信中に `FormHandler` が回復不能な問題に遭遇した場合（例: ネットワークエラー）にエラーをスローします。
Auth0 からの検証エラー（無効なニックネームなど）は JavaScript のエラーとしてスローされず、
処理後に `this.transaction.errors` に反映されます。

```typescript Example
// 'sdk' が MfaWebAuthnChangeKeyNickname のインスタンスであると仮定します
const newNickname = "My Favorite YubiKey"; // ユーザー入力の値
try {
  await sdk.continueWithNewNickname({ nickname: newNickname });
  // 成功した場合、ページはリダイレクトされます。
} catch (error) {
  // この catch は SDK 呼び出し自体の予期しないエラー用です。
  console.error("Failed to submit the new nickname:", error);
}
// await の後は常に、サーバー側の検証問題がないか sdk.transaction.errors を確認してください。
if (sdk.transaction.errors && sdk.transaction.errors.length > 0) {
  sdk.transaction.errors.forEach(err => {
    if (err.field === 'nickname') {
      // ニックネーム入力フィールドに関連する err.message を表示します。
      // 例: "Name is too short", "Name is required"
    }
  });
}
```
