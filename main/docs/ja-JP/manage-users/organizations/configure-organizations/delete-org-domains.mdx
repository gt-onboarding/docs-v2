<Warning>
  Organization Discovery ドメインは現在、[Early Access](/docs/ja-JP/troubleshoot/product-lifecycle/product-release-stages) として提供されています。この機能を使用することで、Okta の [Master Subscription Agreement](https://www.okta.com/legal/) に定められた該当する無料トライアル条件に同意したものとみなされます。
</Warning>

Organization のディスカバリードメインを削除すると、そのドメインと組織との関連付けは恒久的に削除されます。

<Warning>
  ディスカバリードメインの削除を確定すると、この操作は元に戻せません。
</Warning>

Auth0 Dashboard または Management API を使用して、Organization ディスカバリードメインを削除できます。

<Tabs>
  <Tab title="Auth0 Dashboard">
    Auth0 Dashboard で Organization ディスカバリードメインを削除するには:

    1. [Auth0 Dashboard &gt; Organizations](https://manage.auth0.com/#/organizations) に移動します。
    2. **Domains** タブを選択します。
    3. **Add Domain** を選択します。
    4. 削除したいドメインを探します。
    5. ドメインの横にあるアクションメニュー (⋮) を開き、**Delete** を選択します。
    6. 削除を確定します。
  </Tab>

  <Tab title="Management API">
    <Callout icon="file-lines" color="#0EA5E9" iconType="regular">
      <b>Auth0 ドメインを確認する</b>

      Auth0 ドメインは、テナント名、リージョンのサブドメイン、および `auth0.com` を組み合わせたものです。ただし、US リージョンで 2020 年 6 月以前に作成されたテナントは例外です。

      例: テナント名が Travel0 でリージョンが US の場合、Auth0 ドメインは `travel0.us.auth0.com` です。US リージョンで 2020 年 6 月以前にテナントが作成されている場合、ドメイン名は `https://travel0.auth0.com` となります。

      Auth0 Dashboard でカスタムドメインを設定および検証済みの場合、ドメインはそのカスタムドメイン名になります。詳細は、[Custom Domains](/docs/ja-JP/customize/custom-domains) を参照してください。
    </Callout>

    Management API の `Delete Organization Discovery Domain` エンドポイントを呼び出します。プレースホルダー値は、テナントに対応する値に必ず置き換えてください。

    ```bash
    curl --request DELETE \
      --url 'https://{yourDomain}/api/v2/organizations/{ORGANIZATION_ID}/discovery-domains/{ORGANIZATION_DOMAIN_ID}' \
      --header 'authorization: Bearer {MGMT_API_ACCESS_TOKEN}'
    ```

    | <b>Value</b> | <b>Description</b> |
    | ----- | ----------- |
    | `MGMT_API_ACCESS_TOKEN` | スコープ `delete:organization` を持つ [Management API 用アクセストークン](/docs/ja-JP/tokens/management-api-access-tokens)。 |
    | `ORGANIZATION_ID` | ディスカバリードメインを追加したい組織の ID (例: `org_abc123`)。 |
    | `ORGANIZATION_DOMAIN_ID` | 必須。Organization Discovery 用に関連付けたいメールドメイン (例: `example.com`)。|

    | <b>ステータスコード</b> | <b>エラーの説明</b> | <b>メッセージ</b> | <b>原因</b> |
    | ------- | ------- | ------- | ------- |
    | `204` |  | ディスカバリードメインが削除されました。 ||
    | `400` | `invalid_uri` | リクエスト URI が無効です。メッセージ内容は原因によって異なります。 | パスが無効です。 |
    | `401` | | トークンが無効です。 ||
    | `401` | | JSON Web Token (JWT) の検証で無効な署名が受信されました。 ||
    | `403` | `insufficient_scope` | スコープが不十分です。次のいずれかが必要です: `delete:organization_discovery_domains`。 | 付与された Bearer トークンのスコープでは許可されていないフィールドを読み取りまたは書き込みしようとしました。 |
    | `404` | | Organization ドメインが見つかりません。 | |
    | `429` |  | リクエストが多すぎます。`X-RateLimit-Limit`、`X-RateLimit-Remaining`、`X-RateLimit-Reset` ヘッダーを確認してください。 ||
  </Tab>
</Tabs>
