    このガイドでは、[go-jwt-middleware](https://github.com/auth0/go-jwt-middleware) パッケージを使用して、新規または既存の Go API アプリケーションに Auth0 を統合する方法を説明します。

    Auth0 Dashboard でまだ API を作成していない場合は、インタラクティブセレクターを使用して新しい Auth0 API を作成するか、プロジェクト用の既存の API を選択してください。

    Auth0 Dashboardで最初のAPIをセットアップするには、入門ガイドを参照してください。

    各Auth0 APIはAPI識別子を使用します。アプリケーションはアクセストークンを検証する際にこの識別子が必要です。

    <Section id={sections[0].id} title={sections[0].title} stepNumber="1" isSingleColumn>
      Permissions を使用すると、特定のアクセストークンを持つユーザーに代わって、リソースへどのようにアクセスできるかを定義できます。
      たとえば、ユーザーが manager アクセスレベルを持っている場合は `messages` リソースへの読み取りアクセスを付与し、administrator アクセスレベルを持っている場合はそのリソースへの書き込みアクセスを付与するといったことができます。

      許可する Permissions は、Auth0 Dashboard の [APIs](https://manage.auth0.com/#/apis) セクションにある **Permissions** ビューで定義できます。次の例では、`read:messages` スコープを使用します。

      ```go Go lines
      // go.mod
      module 01-Authorization-RS256
      go 1.21
      require (
      github.com/auth0/go-jwt-middleware/v2 v2.2.0

      github.com/joho/godotenv v1.5.1

      )
      ```

      次のシェル コマンドを実行して、依存関係をダウンロードします：


      ```env lines

      ```

    <Section id={sections[3].id} title={sections[3].title} stepNumber="4" isSingleColumn>
      `EnsureValidToken` ミドルウェア関数は、アクセストークンを検証します。この関数は、保護したい任意のエンドポイントに適用できます。トークンが有効な場合、エンドポイントはリソースへのアクセスを許可します。トークンが無効な場合、API は `401 Authorization` エラーを返します。

      受信リクエストのアクセストークンを検証するために、**go-jwt-middleware** ミドルウェアをセットアップします。

      デフォルトでは、API はトークン署名アルゴリズムとして RS256 を使用するよう設定されています。RS256 は秘密鍵/公開鍵ペアを使用して動作するため、トークンは Auth0 アカウントの公開鍵に対して検証できます。この公開鍵は `https://{yourDomain}/.well-known/jwks.json` から取得できます。

      要求されたリソースへアクセスするために、トークンが十分な **スコープ** を持っているか確認する仕組みを用意してください。

      成功レスポンスを返す前に、アクセストークンが正しいスコープを持っていることを確認するための関数 `HasScope` を作成します。

    <Section id={sections[4].id} title={sections[4].title} stepNumber="5" isSingleColumn>
      この例では、認証されていないリクエストにアクセス可能とするため、`EnsureToken`ミドルウェアを使用しない`/api/public`エンドポイントを作成します。

      `/api/private` エンドポイントを作成します。このエンドポイントは `EnsureToken` ミドルウェアを必要とし、追加のスコープを含まないアクセストークンを持つ認証済みリクエストのみが利用可能です。

      `/api/private-scoped` エンドポイントを作成します。このエンドポイントは `EnsureToken` ミドルウェアと `HasScope` を必要とし、`read:messages` スコープが付与されたアクセストークンを含む認証済みリクエストでのみ利用可能です。

      <Info>
        `HasScope` 関数によってチェックされるのは `read:messages` スコープのみです。ユースケースに合わせて、これを拡張するか、複数のスコープを受け付けるスタンドアロンのミドルウェアにすることを検討してください。
      </Info>

        ##### チェックポイント

        アプリケーションの設定が完了したら、アプリケーションを実行し、次の点を確認してください。
